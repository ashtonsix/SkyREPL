================================================================================
8. PROVIDERS
================================================================================

## Purpose

The provider system abstracts cloud infrastructure operations (spawn, terminate,
snapshot) across platforms while allowing platform-specific optimizations.
Providers must satisfy logical contracts (intents) defining required
outputs, but have significant implementation freedom.

**Key principles:**
- Providers are NOT uniform. Complex platforms (AWS, GCP, Azure) benefit from
  deep integration; simpler providers use fixed, minimal workflows.
- The control plane cares about outcomes ("I need an instance"), not
  implementation details.
- 80% of functionality works across providers through the common interface.
  The remaining 20% can be platform-specific where justified.
- Testing against OrbStack (with fault injection) validates contract-compliant
  behavior; providers following the same contracts inherit that validation.

## 8.1 Provider Interface

### Spawn Idempotency Design

Instance spawning uses a two-phase protocol to guarantee idempotency and prevent
resource orphaning, even with crashes or network failures.

**Design Principles:**
1. DB record created BEFORE cloud API call
2. Instance added to manifest BEFORE cloud API call
3. Reconciliation handles orphaned spawn:pending records
4. Property test satisfied: "Kill at any step → no orphans"

**Phase 1: Reserve in DB**
```typescript
async function reserveInstance(ctx: NodeContext, options: SpawnOptions): Promise<Instance> {
  const idempotencyKey = `${ctx.workflowId}:${ctx.nodeId}`;

  // Create instance record with spawn:pending BEFORE calling provider
  // Note: ID auto-generated by database (INTEGER PRIMARY KEY AUTOINCREMENT)
  const result = await db.execute(`
    INSERT INTO instances (
      provider, provider_id, spec, region, workflow_state,
      spawn_idempotency_key, current_manifest_id, created_at, last_heartbeat
    ) VALUES (?, ?, ?, ?, 'spawn:pending', ?, ?, ?, ?)
  `, [
    options.provider,
    '', // provider_id not yet assigned
    options.spec,
    options.region,
    idempotencyKey,
    ctx.manifestId,
    Date.now(),
    Date.now()
  ]);

  const instanceId = result.lastInsertRowid;

  // Add to manifest BEFORE calling provider
  await ctx.emitResource('instance', instanceId);

  return { id: instanceId, ...options, workflow_state: 'spawn:pending' };
}
```

**Phase 2: Execute Spawn**
```typescript
async function executeSpawn(
  ctx: NodeContext,
  instance: Instance,
  options: SpawnOptions
): Promise<Instance> {
  const provider = getProvider(options.provider);

  // Call provider.spawn() with clientRequestId if supported
  const providerOptions = {
    ...options,
    clientRequestId: provider.capabilities.idempotentSpawn
      ? instance.spawn_idempotency_key
      : undefined
  };

  try {
    const providerInstance = await provider.spawn(providerOptions);

    // Update DB with provider_id and transition to provisioning
    await db.execute(`
      UPDATE instances
      SET provider_id = ?,
          workflow_state = ?,
          ip = ?,
          last_heartbeat = ?
      WHERE id = ?
        AND workflow_state = 'spawn:pending'
    `, [
      providerInstance.id,
      mapProviderStatus(providerInstance.status),
      providerInstance.ip,
      Date.now(),
      instance.id
    ]);

    return { ...instance, provider_id: providerInstance.id, workflow_state: 'launch-run:provisioning' };
  } catch (error) {
    // On error, mark as failed but keep record for retry/reconciliation
    await db.execute(`
      UPDATE instances SET workflow_state = 'spawn:error', last_heartbeat = ?
      WHERE id = ? AND workflow_state = 'spawn:pending'
    `, [Date.now(), instance.id]);
    throw error;
  }
}
```

**Retry Behavior:**
```typescript
async function retrySpawn(ctx: NodeContext, failedInstanceId: string): Promise<Instance> {
  const instance = await db.queryOne(`
    SELECT * FROM instances WHERE id = ?
  `, [failedInstanceId]);

  if (instance.workflow_state === 'spawn:pending') {
    // Check with provider if instance was actually created
    const reconciled = await reconcilePendingSpawn(instance);
    if (reconciled) {
      return reconciled; // Found the instance in provider, no retry needed
    }
  }

  // Clean up failed attempt
  await db.execute(`
    UPDATE instances SET workflow_state = 'spawn:error' WHERE id = ?
  `, [failedInstanceId]);

  // Create new attempt (Phase 1 again)
  const newInstance = await reserveInstance(ctx, instance);
  return await executeSpawn(ctx, newInstance, instance);
}
```

**Reconciliation (Orphan Detection):**
```typescript
async function reconcilePendingSpawn(instance: Instance): Promise<Instance | null> {
  const provider = getProvider(instance.provider);

  // For providers that support clientRequestId, query by that
  if (provider.capabilities.idempotentSpawn) {
    const providerInstances = await provider.list({
      tags: { 'skyrepl:spawn_key': instance.spawn_idempotency_key }
    });

    if (providerInstances.length > 0) {
      // Found the instance! Update our record
      const providerInstance = providerInstances[0];
      await db.execute(`
        UPDATE instances
        SET provider_id = ?, workflow_state = ?, ip = ?, last_heartbeat = ?
        WHERE id = ?
      `, [
        providerInstance.id,
        mapProviderStatus(providerInstance.status),
        providerInstance.ip,
        Date.now(),
        instance.id
      ]);
      return { ...instance, provider_id: providerInstance.id };
    }
  }

  // Instance not found in provider - truly failed
  return null;
}

// Periodic reconciliation finds stale spawn:pending records
async function reconcileStalePendingSpawns(): Promise<void> {
  const staleThresholdMs = timing.PENDING_SPAWN_STALE_THRESHOLD_MS;
  const staleInstances = await db.query(`
    SELECT * FROM instances
    WHERE workflow_state = 'spawn:pending'
      AND created_at < ?
  `, [Date.now() - staleThresholdMs]);

  for (const instance of staleInstances) {
    const reconciled = await reconcilePendingSpawn(instance);
    if (!reconciled) {
      // Mark as failed after reconciliation timeout
      await db.execute(`
        UPDATE instances SET workflow_state = 'spawn:error' WHERE id = ?
      `, [instance.id]);
    }
  }
}
```

**Compensation (Single-Node Cleanup):**
```typescript
async function compensateSpawn(ctx: CompensationContext): Promise<void> {
  const instance = ctx.output as Instance;

  // If we made it to Phase 2 (have provider_id), terminate in provider
  if (instance.provider_id) {
    const provider = getProvider(instance.provider);
    try {
      await provider.terminate(instance.provider_id);
    } catch (error) {
      // Instance may already be terminated, that's OK
      if (error.code !== 'NOT_FOUND') throw error;
    }
  }
  // If provider_id is null, Phase 1 completed but Phase 2 crashed.
  // Skip provider.terminate() since no cloud resource exists.
  // Tag-based orphan recovery (Ch12) is the primary cleanup mechanism.

  // Mark as terminated in DB
  await db.execute(`
    UPDATE instances SET workflow_state = 'terminate:complete' WHERE id = ?
  `, [instance.id]);
}
```

**Kill-at-any-step Analysis:**
- **Kill before Phase 1**: Nothing created, transaction not committed
- **Kill during Phase 1**: If transaction commits, we have DB record with spawn:pending
  but no cloud resource. Reconciliation detects and marks as failed.
- **Kill after Phase 1, before Phase 2**: DB has spawn:pending, no cloud resource.
  Reconciliation marks as failed.
- **Kill during Phase 2**: Two sub-cases:
  - Provider call succeeds but UPDATE fails: Reconciliation finds instance by
    clientRequestId (if supported) or via periodic list() sweep, updates DB record
  - Provider call fails: No cloud resource created, DB record remains spawn:pending,
    reconciliation marks as failed
- **Kill after Phase 2**: Normal completion, no orphans

**Provider Capability Flag:**
```typescript
interface ProviderCapabilities {
  // ... existing flags

  /** Provider supports idempotent spawn via clientRequestId */
  idempotentSpawn: boolean;
}
```

For providers that don't support clientRequestId (Lambda Labs, RunPod), reconciliation
relies on periodic list() operations and matching by tags/timestamps. Less efficient
but still prevents orphans.

### Core Interface Definition

```typescript
/**
 * Provider interface defines the contract all providers must implement.
 * Core methods are required; optional capabilities are gated by flags.
 *
 * CONTRACTS: Provider contracts are enforced via TypeScript types + unit tests.
 * Every provider MUST pass interface compliance tests that verify correct behavior
 * (spawn returns required fields, terminate handles non-existent instances, etc.).
 * This is not runtime contract checking in production, just good tests that ensure
 * providers satisfy the logical contract.
 */
interface Provider<
  TInstance extends ProviderInstance = ProviderInstance,
  TSnapshot extends ProviderSnapshot = ProviderSnapshot
> {
  /** Provider identifier (aws, lambda, runpod, orbstack) */
  readonly name: ProviderName;

  /** Capability flags for optional features */
  readonly capabilities: ProviderCapabilities;

  // ============================================================
  // CORE METHODS (Required)
  // ============================================================

  /**
   * Spawn a new instance with the given configuration.
   * @returns Provider-specific instance with at minimum { id, status }
   */
  spawn(options: SpawnOptions): Promise<TInstance>;

  /**
   * Terminate an instance.
   * @param providerId - Provider's instance ID (i-xxx, etc.)
   * @throws ProviderError if instance not found or already terminated
   */
  terminate(providerId: string): Promise<void>;

  /**
   * List instances, optionally filtered by status.
   * Used for reconciliation and drift detection.
   */
  list(filter?: ListFilter): Promise<TInstance[]>;

  /**
   * Get a single instance by provider ID.
   * @returns Instance or null if not found
   */
  get(providerId: string): Promise<TInstance | null>;

  /**
   * Generate cloud-init/bootstrap script for instance initialization.
   * Encapsulates all provider-specific boot configuration.
   */
  generateBootstrap(config: BootstrapConfig): BootstrapScript;

  // ============================================================
  // OPTIONAL METHODS (Gated by capabilities)
  // ============================================================

  /**
   * Create a snapshot of an instance.
   * Only available if capabilities.snapshots === true
   */
  createSnapshot?(
    providerId: string,
    options: CreateSnapshotOptions
  ): Promise<SnapshotRequest>;

  /**
   * Get snapshot creation status.
   * Only available if capabilities.snapshots === true
   */
  getSnapshotStatus?(requestId: string): Promise<SnapshotStatus>;

  /**
   * Delete a snapshot.
   * Only available if capabilities.snapshots === true
   */
  deleteSnapshot?(providerSnapshotId: string): Promise<void>;

  /**
   * Get snapshot by name.
   * Only available if capabilities.snapshots === true
   */
  getSnapshotByName?(name: string): Promise<TSnapshot | null>;

  /**
   * Request spot instance.
   * Only available if capabilities.spot === true
   */
  requestSpotInstance?(options: SpotOptions): Promise<SpotRequest>;

  /**
   * Cancel spot request.
   * Only available if capabilities.spot === true
   */
  cancelSpotRequest?(requestId: string): Promise<void>;

  /**
   * Get current spot prices.
   * Only available if capabilities.spot === true
   */
  getSpotPrices?(spec: string, regions?: string[]): Promise<SpotPriceInfo[]>;
}

// Provider names as const union
type ProviderName = 'aws' | 'lambda' | 'runpod' | 'orbstack';

/**
 * CONTRACT TESTING: Every provider must pass these tests.
 *
 * Example test concepts (actual tests in provider.test.ts):
 *
 * test('spawn returns required fields', async () => {
 *   const result = await provider.spawn(validOptions);
 *   expect(result).toHaveProperty('id');
 *   expect(result).toHaveProperty('status');
 *   expect(result.status).toMatch(/pending|starting|running/);
 * });
 *
 * test('terminate handles non-existent instance', async () => {
 *   await expect(provider.terminate('fake-id'))
 *     .rejects.toThrow(NotFoundError);
 * });
 *
 * test('list filters by region', async () => {
 *   const results = await provider.list({ region: 'us-east-1' });
 *   expect(results.every(i => i.region?.startsWith('us-east-1'))).toBe(true);
 * });
 *
 * test('generateBootstrap returns valid script', async () => {
 *   const script = provider.generateBootstrap(config);
 *   expect(script.content).toBeTruthy();
 *   expect(script.format).toMatch(/cloud-init|shell|ignition/);
 *   expect(script.checksum).toHaveLength(64); // SHA-256
 * });
 */
```

### Capability Flags

Capabilities describe what optional features a provider supports. The control
plane uses these to gate optional method calls.

**Capability groups:**

| Group    | Capabilities                                                    |
|----------|-----------------------------------------------------------------|
| COMMON   | spawn, terminate, list, get, generateBootstrap (all required)   |
| PRIORITY | snapshots, spot, gpu, multiRegion, persistentVolumes            |
| SPECIAL  | warmVolumes, hibernation, costExplorer, tailscaleNative,        |
|          | idempotentSpawn, customNetworking                               |

Note: Lines blur between tiers at higher capability levels. A provider may
implement special capabilities without implementing all priority ones.

```typescript
interface ProviderCapabilities {
```

### Error Taxonomy

```typescript
/**
 * Base class for all provider errors.
 * Provides structured error information for handling.
 */
abstract class ProviderError extends Error {
  abstract readonly code: ProviderErrorCode;
  abstract readonly category: ProviderErrorCategory;
  abstract readonly retryable: boolean;
  abstract readonly retryAfterMs?: number;

  constructor(
    message: string,
    public readonly provider: ProviderName,
    public readonly details?: Record<string, unknown>
  ) {
    super(message);
    this.name = this.constructor.name;
  }
}

type ProviderErrorCode =
  | 'CAPACITY_ERROR'
  | 'AUTH_ERROR'
  | 'RATE_LIMIT_ERROR'
  | 'SPOT_INTERRUPTED'
  | 'QUOTA_EXCEEDED'
  | 'INVALID_SPEC'
  | 'REGION_UNAVAILABLE'
  | 'NETWORK_ERROR'
  | 'TIMEOUT_ERROR'
  | 'NOT_FOUND'
  | 'ALREADY_EXISTS'
  | 'INVALID_STATE'
  | 'PROVIDER_INTERNAL'
  | 'UNSUPPORTED_OPERATION';

type ProviderErrorCategory =
  | 'capacity'     // Resource availability issues
  | 'auth'         // Authentication/authorization
  | 'rate_limit'   // Rate limiting / throttling

  | 'conflict'     // State conflict
  | 'internal';    // Provider internal error

// ERROR HANDLING HELPERS
// ============================================================

/**
 * Map provider-specific errors to our error taxonomy.
 */
function mapProviderError(
  provider: ProviderName,
  error: unknown
): ProviderError {
  // Provider-specific mapping happens in provider implementations
  // This is the fallback for unmapped errors
  if (error instanceof ProviderError) return error;

  const message = error instanceof Error ? error.message : String(error);
  return new class extends ProviderError {
    readonly code = 'PROVIDER_INTERNAL' as const;
    readonly category = 'internal' as const;
    readonly retryable = false;
  }(message, provider, { originalError: error });
}

/**
 * Determine if error should trigger Retry-with-Alternative pattern.
 */
function shouldRetryWithAlternative(error: ProviderError): boolean {
  return (
    error.code === 'CAPACITY_ERROR' ||
    error.code === 'SPOT_INTERRUPTED' ||
    error.code === 'REGION_UNAVAILABLE'
  );
}

/**
 * Determine if error should trigger simple retry.
 */
function shouldRetry(error: ProviderError): boolean {
  return error.retryable && error.category !== 'auth';
}
```

```typescript
        '/usr/local/bin/skyrepl-agent start',

      content,
      format: 'cloud-init',
```

### Materialization Pattern (Generic -> Provider-Specific)

```typescript
/**
 * MATERIALIZATION: When workflow enters provider context,
 * generic Instance is enhanced to provider-specific type.
 *
 * Happens at provider-specific workflow/subworkflow boundaries.
 */

async function materializeAWSInstance(
  genericInstance: Instance
): Promise<AWSInstance> {
  if (genericInstance.provider !== 'aws') {
    throw new Error(`Cannot materialize non-AWS instance as AWSInstance`);
  }

  // Fetch AWS-specific data
  const ec2 = new EC2Client({});

  const describeResult = await ec2.send(new DescribeInstancesCommand({
    InstanceIds: [genericInstance.providerId],
  }));

  const ec2Instance = describeResult.Reservations?.[0]?.Instances?.[0];
  if (!ec2Instance) {
    throw new NotFoundError('aws', 'instance', genericInstance.providerId);
  }

  // Build enhanced instance
  const awsInstance: AWSInstance = {
    // Base provider fields
    id: genericInstance.providerId,
    status: mapEC2Status(ec2Instance.State?.Name),
    spec: ec2Instance.InstanceType ?? genericInstance.spec,
    region: genericInstance.region,
    ip: ec2Instance.PublicIpAddress ?? null,
    privateIp: ec2Instance.PrivateIpAddress ?? null,
    createdAt: ec2Instance.LaunchTime?.getTime() ?? genericInstance.createdAt,
    isSpot: ec2Instance.InstanceLifecycle === 'spot',
    spotRequestId: ec2Instance.SpotInstanceRequestId,

    // AWS-specific fields
    availabilityZone: ec2Instance.Placement?.AvailabilityZone ?? '',
    subnetId: ec2Instance.SubnetId ?? '',
    vpcId: ec2Instance.VpcId ?? '',
    securityGroupIds: ec2Instance.SecurityGroups?.map(sg => sg.GroupId!) ?? [],
    iamInstanceProfile: ec2Instance.IamInstanceProfile?.Arn,
    ebsOptimized: ec2Instance.EbsOptimized ?? false,

    volumes: ec2Instance.BlockDeviceMappings?.map(bdm => ({
      volumeId: bdm.Ebs?.VolumeId ?? '',
      deviceName: bdm.DeviceName ?? '',
      sizeGb: 0,  // Would need separate DescribeVolumes call
      volumeType: 'gp3',
      encrypted: false,
      deleteOnTermination: bdm.Ebs?.DeleteOnTermination ?? true,
    })) ?? [],

    networkInterfaces: ec2Instance.NetworkInterfaces?.map(ni => ({
      networkInterfaceId: ni.NetworkInterfaceId ?? '',
      subnetId: ni.SubnetId ?? '',
      privateIp: ni.PrivateIpAddress ?? '',
      publicIp: ni.Association?.PublicIp,
      securityGroupIds: ni.Groups?.map(g => g.GroupId!) ?? [],
    })) ?? [],

    placementGroup: ec2Instance.Placement?.GroupName,

    metadata: {
      launchTemplateId: ec2Instance.LaunchTemplate?.LaunchTemplateId,
      launchTemplateVersion: ec2Instance.LaunchTemplate?.Version,
      capacityReservationId: ec2Instance.CapacityReservationId,
    },
  };

  return awsInstance;
}

    case 'stopping': return 'stopping';
    case 'stopped': return 'stopped';
```

### Projection Pattern (Provider-Specific -> Generic)

```typescript
/**
 * PROJECTION: When workflow exits provider context,
 * provider-specific instance projects back to generic type.
 *
 * Happens when provider workflow completes and resources
 * return to control plane ownership.
 */

function projectToGenericInstance(
  awsInstance: AWSInstance,
  existingRecord?: Instance
): Instance {
  return {
    // Identity (preserve from existing - ID should already exist in DB)
    // Note: This is a projection function, not an insert. The ID must come from existingRecord.
    // If existingRecord is undefined, this indicates a bug - we should not project without a DB record.
    id: existingRecord?.id ?? (() => { throw new Error('Cannot project instance without existing DB record'); })(),
    provider: 'aws',
    providerId: awsInstance.id,

    // Spec and location
    spec: awsInstance.spec,
    region: awsInstance.availabilityZone?.slice(0, -1),  // us-east-1a -> us-east-1

    // Network
    ip: awsInstance.ip,
    tailscaleIp: existingRecord?.tailscaleIp,  // Preserve from existing

    // Workflow state (map to workflow-aware format)
    workflow_state: existingRecord?.workflow_state ?? mapStatusToWorkflowState(awsInstance.status),

    // Spot
    isSpot: awsInstance.isSpot,
    spotRequestId: awsInstance.spotInstanceRequestId,

    // Init checksum (preserve from existing)
    initChecksum: existingRecord?.initChecksum,

    // Timestamps
    createdAt: awsInstance.createdAt,
    lastHeartbeat: existingRecord?.lastHeartbeat ?? Date.now(),

    // Ownership
    currentManifestId: existingRecord?.currentManifestId,
  };
}

async function projectToGenericSnapshot(awsSnapshot: AWSSnapshot): Promise<Snapshot> {
  // Note: Snapshot IDs are auto-generated by database when the snapshot record is created.
  // This projection function should only be called AFTER inserting the snapshot into the DB.
  // The caller must insert first and get the auto-generated ID, then use that ID.
  // This function is used for returning snapshot data, not creating new records.
  // If you need to create a snapshot, use the appropriate insert query first.

  // For projection purposes where we need a snapshot object but don't have a DB record yet,
  // this indicates the caller should insert first then retrieve the auto-generated ID.
  throw new Error('projectToGenericSnapshot should not generate IDs - insert snapshot record first to get auto-generated ID');
}

// Utility for workflow_state mapping from provider status
function mapStatusToWorkflowState(providerStatus: ProviderInstanceStatus): string {
  switch (providerStatus) {
    case 'pending':
    case 'starting':
      return 'launch-run:provisioning';
    case 'running':
      return 'launch-run:complete';  // Availability is inferred, not stored
    case 'stopping':
    case 'stopped':
      return 'terminate:draining';
    case 'terminating':
    case 'terminated':
      return 'terminate:complete';
    default:
      return 'spawn:error';
  }
}
```

```typescript
/**
 * Example: AWS warm-volume-pool workflow uses enhanced types internally,
 * projects to generic on completion.
 */

// Inside AWS provider workflow
async function awsWarmVolumePoolWorkflow(ctx: NodeContext): Promise<WorkflowOutput> {
  // ENTER: Materialize to AWS-specific type
  const awsInstance = await materializeAWSInstance(ctx.input.instance);

  // Internal AWS-specific operations using enhanced type
  const volume = await createWarmVolume({
    availabilityZone: awsInstance.availabilityZone,
    snapshotId: ctx.input.snapshotId,
    volumeType: 'gp3',
    iops: 10000,
    throughput: 250,
  });

  await attachVolume({
    instanceId: awsInstance.id,
    volumeId: volume.volumeId,
    device: '/dev/sdf',
  });

  // Update local state
  awsInstance.volumes.push(volume);

  // EXIT: Project back to generic for control plane
  const genericInstance = projectToGenericInstance(awsInstance, ctx.input.instance);

  // Emit to manifest with generic type
  ctx.emitResource('instance', genericInstance.id);

  return {
    instance: genericInstance,
    volumeId: volume.volumeId,
  };
}
```

## 8.3 Provider Workflows

Providers can define workflow types to encapsulate complex
provider-specific logic. Provider workflows must satisfy abstract contracts
for uniform control plane interaction.

### Constraints on Provider-Defined Workflows

```typescript
/**
 * Provider workflows have constraints ensuring
 * uniform control plane management.
 */

interface ProviderWorkflowConstraints {
  /** Maximum workflow duration */
  maxDurationMs: number;

  /** Required output fields */
  requiredOutputFields: string[];

  /** Allowed input types */
  inputSchema: JSONSchema;

  /** Resources the workflow may emit */
  emittableResources: ResourceType[];

  /** Whether workflow can spawn subworkflows */
  canSpawnSubworkflows: boolean;

  /** Maximum subworkflow depth */
  maxSubworkflowDepth: number;
}

/**
 * Abstract contracts that provider workflows must satisfy.
 */
interface ProviderWorkflowContract {
  /** Workflow type identifier */
  type: string;

  /** Provider this workflow belongs to */
  provider: ProviderName;

  /** Description of what this workflow does */
  description: string;

  /** Constraints */
  constraints: ProviderWorkflowConstraints;

  /**
   * Validation function for input.
   * Called before workflow submission.
   */
  validateInput(input: unknown): ValidationResult;

  /**
   * Compensation function for single-node cleanup.
   * Called when this node fails and needs cleanup.
   */
  compensate(context: CompensationContext): Promise<void>;
}

/**
 * Registry of provider workflow types.
 */
const PROVIDER_WORKFLOW_REGISTRY: Map<string, ProviderWorkflowContract> = new Map();

function registerProviderWorkflow(contract: ProviderWorkflowContract): void {
  const key = `${contract.provider}:${contract.type}`;
  if (PROVIDER_WORKFLOW_REGISTRY.has(key)) {
    throw new Error(`Provider workflow already registered: ${key}`);
  }
  PROVIDER_WORKFLOW_REGISTRY.set(key, contract);
}
```

```typescript
    WHERE type = 'warm_volume'

async function createWarmVolumes(ctx: NodeContext): Promise<{ volumes: WarmVolume[] }> {
  const { count, availabilityZone, snapshotId, volumeSpec } = ctx.input;

  const ec2 = new EC2Client({});
  const volumes: WarmVolume[] = [];

  // Fan-out volume creation
  for (let i = 0; i < count; i++) {
    const result = await ec2.send(new CreateVolumeCommand({
      AvailabilityZone: availabilityZone,
      SnapshotId: snapshotId,
      VolumeType: volumeSpec?.type ?? 'gp3',

      Throughput: volumeSpec?.throughput ?? 125,

          { Key: 'skyrepl:snapshot-id', Value: snapshotId },

      snapshotId,

    const result = await db.query(`

        status: 'available',

      Date.now(),
      Date.now(),
```

```typescript
    const result = await waitForWorkflow(subworkflow.id);

    // Result satisfies SpawnInstanceContract regardless of provider
    return result.output as SpawnInstanceOutput;
  } else {
    // Use standard spawn
    const instance = await provider.spawn(runInput);
    return projectToGenericInstance(instance);
  }
}
```

## 8.4 Feature Providers

Feature providers attach cross-cutting concerns to existing instances. Unlike
compute providers (AWS, Lambda, RunPod, OrbStack) creating and destroying compute,
feature providers enhance instances with capabilities:
networking (Tailscale), logging (WandB), remote development (VS Code Server).

### FeatureProvider Interface

```typescript
/**
 * Feature providers attach to existing instances rather than creating compute.
 * They enable cross-cutting concerns like networking, logging, and development tools.
 */
interface FeatureProvider<TConfig, TState> {
  /** Feature provider name (tailscale, wandb, vscode-server) */
  name: string;

  /** Feature capabilities */
  capabilities: FeatureCapabilities;

  /**
   * Attach feature to an instance.
   * Called during instance spawn or hot-attach after running.
   */
  attach(instanceId: string, config: TConfig): Promise<TState>;

  /**
   * Detach feature from instance.
   * Called during instance termination or feature removal.
   */
  detach(instanceId: string): Promise<void>;

  /**
   * Get current feature state for an instance.
   * Returns null if feature not attached or state unavailable.
   */
  status(instanceId: string): Promise<TState | null>;

  /**
   * Optional: Reconcile feature state during orphan detection.
   * Called during orphan detection cycle to find feature-specific orphans.
   * Example: Tailscale checks for orphaned machines in tailnet.
   * Returns list of orphaned feature resources for cleanup consideration.
   */
  reconcile?(context: FeatureReconcileContext): Promise<ReconcileResult>;

  /**
   * Optional: Called on instance heartbeat for feature maintenance.
   */
  onInstanceHeartbeat?(instanceId: string): Promise<void>;
}

interface FeatureCapabilities {
  /** Feature requires bootstrap script injection */
  requiresBootstrap: boolean;

  /** Feature can attach after instance already running */
  supportsHotAttach: boolean;

  /** Feature can detach without terminating instance */
  supportsDetach: boolean;

  /** Feature works with all compute providers */
  crossProvider: boolean;
}

interface FeatureReconcileContext {
  /** Instances that should have this feature attached */
  expectedAttachments: Array<{
    instanceId: string;
    config: unknown;
  }>;

  /** Current feature state from provider */
  actualState: Map<string, unknown>;
}
```

### Feature vs Compute Providers

```
COMPUTE PROVIDERS              FEATURE PROVIDERS
----------------------------------------------------------------------
Create/destroy compute         Attach to existing instances
Instance lifecycle primary     Instance lifecycle secondary
Direct cloud API interaction   Often API + agent coordination
One per instance               Multiple per instance
Examples: AWS, Lambda, RunPod  Examples: Tailscale, WandB, VS Code
```

## 8.5 Extension Framework

The extension framework provides hooks and storage for provider-specific
behavior without polluting core control plane code.

### Provider State (In-Memory Cache)

Providers cache ephemeral data (spot prices, capacity info) in memory with TTL.
This data is not persisted—re-fetched on control plane restart.

```typescript
// Simple cache wrapper
async function getWithCache<T>(
  key: string,
  ttlMs: number,
  fetcher: () => Promise<T>
): Promise<T> {
  const cached = cache.get(key);
  if (cached && Date.now() < cached.expiresAt) {
    return cached.value;
  }
  const value = await fetcher();
  cache.set(key, { value, expiresAt: Date.now() + ttlMs });
  return value;
}

// Usage
const spotPrices = await getWithCache(
  `aws:spot_prices:${region}:${spec}`,
  300_000, // 5 min TTL
  () => ec2.getSpotPrices(spec, region)
);
```

No database table needed. Cache is cleared on restart, which is fine for ephemeral provider state.

### Lifecycle Hooks: Heartbeat Expectations Model

Provider lifecycle hooks use an expectations-based model where the control plane
explicitly communicates what it needs and providers respond with receipts of
completed work.

```typescript
/**
 * Lifecycle hooks let providers respond to system events.
 * Core sends expectations, provider returns receipts.
 */

interface ProviderLifecycleHooks {
  /**
   * Called when control plane starts.
   * For initialization, cache warming, health checks.
   */
  onStartup?(): Promise<void>;

  /**
   * Called on graceful shutdown.
   * For cleanup, state persistence.
   */
  onShutdown?(): Promise<void>;

  /**
   * Unified heartbeat with expectations model.
   * Core sends what it needs, provider responds with what it did.
   */
  onHeartbeat?(expectations: HeartbeatExpectations): Promise<HeartbeatReceipts>;
}

/**
 * HeartbeatExpectations: Core tells provider which tasks to perform.
 */
interface HeartbeatExpectations {
  /** Tasks for provider to execute */
  tasks: HeartbeatTask[];

  /** Timestamp by which tasks should complete */
  deadline: number;
}

interface HeartbeatTask {
  /** Task type */
  type: 'reconcile' | 'health_check' | 'cache_refresh' | 'pool_maintenance';

  /** Task priority */
  priority: 'high' | 'normal' | 'low';

  /** Last time this task ran (for scheduling) */
  lastRun?: number;

  /** Task-specific parameters */
  params?: Record<string, unknown>;
}

/**
 * HeartbeatReceipts: Provider reports what it accomplished.
 */
interface HeartbeatReceipts {
  /** Receipts for requested tasks */
  receipts: TaskReceipt[];

  /** Provider-initiated tasks (not requested by core) */
  providerTasks?: ProviderTaskReport[];

  /**
   * Workflows launched for async fulfillment.
   *
   * Some tasks (like spot interruption handling or budget threshold responses)
   * require workflow orchestration rather than synchronous completion. When a
   * heartbeat task or external event triggers workflow creation, report it here.
   *
   * This enables the core to track workflows launched by providers without
   * requiring separate event channels. Most heartbeats leave this empty.
   */
  workflowsLaunched?: WorkflowLaunchReport[];

  /** Provider's preferred next heartbeat timing */
  nextHeartbeat?: {
    preferredInterval: number;
  };
}

interface WorkflowLaunchReport {
  /** Workflow ID returned from submission */
  workflowId: string;

  /** Workflow type (e.g., 'handle-spot-interruption', 'budget-cleanup') */
  workflowType: string;

  /** What triggered this workflow */
  trigger: 'spot_interruption' | 'budget_threshold' | 'drift_detected' | 'provider_event';

  /** Additional context */
  context?: Record<string, unknown>;
}

interface TaskReceipt {
  /** Task type (matches HeartbeatTask.type) */
  type: string;

  /** Task completion status */
  status: 'completed' | 'skipped' | 'deferred' | 'failed';

  /** Task result data */
  result?: unknown;

  /** Reason for skip/defer/failure */
  reason?: string;
}

interface ProviderTaskReport {
  /** Provider-specific task type */
  type: string;

  /** Task status */
  status: 'completed' | 'failed';

  /** Task result summary */
  result?: unknown;
}
```

### Heartbeat Flow Example

```typescript
// Core sends expectations
const expectations: HeartbeatExpectations = {
  tasks: [
    { type: 'health_check', priority: 'high' },
    { type: 'reconcile', priority: 'normal', lastRun: 1706900000000 },
    { type: 'cache_refresh', priority: 'low' },
  ],
  deadline: Date.now() + 5000,
};

const receipts = await provider.onHeartbeat(expectations);

// Provider returns receipts
{
  receipts: [
    {
      type: 'health_check',
      status: 'completed',
      result: { healthy: true, latency: 45 }
    },
    {
      type: 'reconcile',
      status: 'completed',
      result: {
        actions: [
          { type: 'update', instanceId: 'i-123', reason: 'IP changed' }
        ]
      }
    },
    {
      type: 'cache_refresh',
      status: 'skipped',
      reason: 'cache_valid'
    }
  ],
  providerTasks: [
    {
      type: 'volume_cleanup',
      status: 'completed',
      result: { count: 2 }
    }
  ],
  nextHeartbeat: { preferredInterval: 30000 }
}
```

### Benefits of Expectations Model

- **Explicit communication**: Core states what it needs, not implicit timing
- **Transparent acknowledgment**: Provider can acknowledge, skip, or defer
- **Health check as first-class**: Health becomes an explicit task
- **Natural reconciliation**: Reconciliation is just another heartbeat task
- **Provider autonomy**: Provider can report its own maintenance work
- **Timing negotiation**: Provider can request preferred heartbeat interval
- **Async fulfillment**: Events requiring workflows (spot interruptions, budget
  thresholds) are reported via `workflowsLaunched` rather than handled inline

```typescript
/**
 * AWS provider lifecycle hooks implementation.

const awsLifecycleHooks: ProviderLifecycleHooks = {
  async onStartup(): Promise<void> {
    debug('AWS/onStartup', { status: 'initializing' });

    // Verify credentials
    const sts = new STSClient({});
    const identity = await sts.send(new GetCallerIdentityCommand({}));
    debug('AWS/onStartup', { accountId: identity.Account });

    await warmSpotPriceCache(['us-east-1', 'us-west-2']);

    // Check and create necessary resources
    await ensureSecurityGroups();
    await ensureIAMRoles();

    debug('AWS/onStartup', { status: 'ready' });
  },

  async onShutdown(): Promise<void> {
    debug('AWS/onShutdown', { status: 'shutting_down' });

    // Persist any in-memory state
    await flushMetricsBuffer();

    // Cancel any pending spot requests (optional - may want to keep)
    // await cancelPendingSpotRequests();

    debug('AWS/onShutdown', { status: 'complete' });
  },

  async onHeartbeat(expectations: HeartbeatExpectations): Promise<HeartbeatReceipts> {
    const receipts: TaskReceipt[] = [];
    const providerTasks: ProviderTaskReport[] = [];

    for (const task of expectations.tasks) {
      switch (task.type) {
        case 'health_check':
          receipts.push(await performHealthCheck());
          break;

        case 'reconcile':
          receipts.push(await performReconciliation(task));

          break;

        case 'pool_maintenance':
          receipts.push(await maintainWarmPools(task));
          break;

        default:
          receipts.push({
            type: task.type,
            status: 'skipped',
            reason: 'unknown_task_type',
          });
      }
    }

    // Provider-initiated maintenance
    const volumeCleanup = await cleanupStaleVolumes();
    if (volumeCleanup.count > 0) {
      providerTasks.push({
        type: 'volume_cleanup',
        status: 'completed',
        result: { count: volumeCleanup.count },
      });
    }

    return {
      receipts,
      providerTasks: providerTasks.length > 0 ? providerTasks : undefined,
      nextHeartbeat: { preferredInterval: 30000 },
    };
  },

/**
 * Heartbeat task handlers
 */

async function performHealthCheck(): Promise<TaskReceipt> {
  const start = Date.now();

  try {
    // Check AWS API connectivity
    const sts = new STSClient({});
    await sts.send(new GetCallerIdentityCommand({}));

    const latency = Date.now() - start;

    return {
      type: 'health_check',
      status: 'completed',
      result: {
        healthy: true,
        latency,
        timestamp: Date.now(),
      },
    };
  } catch (error) {
    return {
      type: 'health_check',
      status: 'failed',
      reason: error instanceof Error ? error.message : 'Unknown error',
    };
  }
}

async function performReconciliation(task: HeartbeatTask): Promise<TaskReceipt> {
  try {
    // Fetch our records
    const instances = await db.query(`
      SELECT * FROM instances WHERE provider = 'aws'
    `);

    // Fetch fresh external state
    const ec2 = new EC2Client({});
    const describeResult = await ec2.send(new DescribeInstancesCommand({}));
    const externalInstances = extractInstancesFromResponse(describeResult);

    // Build lookup maps
    const externalById = new Map(
      externalInstances.map(i => [i.id, i])
    );
    const ourByProviderId = new Map(
      instances.map(i => [i.providerId, i])
    );

    const actions = [];

    // Check for orphans (report only, don't terminate)
    for (const external of externalInstances) {
      if (!ourByProviderId.has(external.id)) {
        debug('RECONCILE/orphan-detected', {
          providerId: external.id,
          provider: 'aws',
          createdAt: external.createdAt,
          ageMs: Date.now() - external.createdAt,
        });
      }
    }

    // Check for missing instances (deleted externally)
    for (const ourInstance of instances) {
      if (!externalById.has(ourInstance.providerId)) {
        await db.query('UPDATE instances SET workflow_state = ? WHERE id = ?', [
          'terminate:complete',
          ourInstance.id,
        ]);
        actions.push({
          type: 'mark_terminated',
          instanceId: ourInstance.id,
          reason: 'Instance deleted externally',
        });
      }
    }

    // Check for drift (IP changes, status differences)
    for (const ourInstance of instances) {
      const external = externalById.get(ourInstance.providerId);
      if (!external) continue;

      const changes: Record<string, any> = {};

      if (external.ip && external.ip !== ourInstance.ip) {
        changes.ip = external.ip;
      }

      const externalMapped = mapProviderStatus(external.status);
      if (externalMapped === 'terminated' && !ourInstance.workflow_state.includes('terminate')) {
        changes.workflow_state = 'terminate:complete';
      }

      if (Object.keys(changes).length > 0) {
        const updates = Object.entries(changes)
          .map(([k, v]) => `${k} = ?`)
          .join(', ');
        await db.query(
          `UPDATE instances SET ${updates} WHERE id = ?`,
          [...Object.values(changes), ourInstance.id]
        );
        actions.push({
          type: 'update',
          instanceId: ourInstance.id,
          reason: 'State drift detected',
          changes,
        });
      }
    }

    return {
      type: 'reconcile',
      status: 'completed',
      result: { actions },
    };
  } catch (error) {
    return {
      type: 'reconcile',
      status: 'failed',
      reason: error instanceof Error ? error.message : 'Unknown error',
    };
  }
}

async function refreshCaches(task: HeartbeatTask): Promise<TaskReceipt> {
  const timeSinceLastRun = task.lastRun ? Date.now() - task.lastRun : Infinity;

  // Skip if cache was refreshed recently

    return {
      type: 'cache_refresh',
      status: 'skipped',
      reason: 'cache_valid',

  } catch (error) {

      type: 'cache_refresh',
      status: 'failed',
      reason: error instanceof Error ? error.message : 'Unknown error',
    };
  }
}

async function onSpotInterruption(notice: SpotInterruptionNotice): Promise<void> {
  debug('AWS/onSpotInterruption', {
    instanceId: notice.instanceId,
    interruptionTime: notice.interruptionTime,
  });

  // Find instance and active allocation
  const instance = await db.queryOne(`
    SELECT i.id, a.id as allocation_id, a.run_id
    FROM instances i
    LEFT JOIN allocations a ON a.instance_id = i.id AND a.status = 'ACTIVE'
    WHERE i.provider_id = ?
  `, [notice.instanceId]);

  if (!instance) return;

  // Mark run as spot-interrupted
  if (instance.run_id) {
    await db.query(`
      UPDATE runs SET spot_interrupted = 1 WHERE id = ?
    `, [instance.run_id]);
  }

  // If we have warning time, try to save state
  if (notice.warningTimeMs && notice.warningTimeMs > 30_000) {
    // Submit checkpoint workflow
    await submitWorkflow({
      type: 'emergency-checkpoint',
      input: {
        instanceId: instance.id,
        allocationId: instance.allocation_id,
        timeRemainingMs: notice.warningTimeMs,
      },
      priority: 'high',
    });
  }

  // Submit retry-with-alternative workflow for the run
  if (instance.run_id) {
    await submitWorkflow({
      type: 'resume-interrupted-run',
      input: {
        runId: instance.run_id,
        interruptedInstanceId: instance.id,
        preferOnDemand: true,
      },
    });
  }
}

async function onBudgetThreshold(threshold: BudgetThreshold): Promise<void> {
  debug('AWS/onBudgetThreshold', {
    thresholdType: threshold.type,
    currentAmount: threshold.currentAmount,
    limitAmount: threshold.limitAmount,
  });

  if (threshold.type === 'warning') {
    // Log warning, don't take action
    console.warn(`AWS budget warning: $${threshold.currentAmount}/$${threshold.limitAmount}`);
  } else if (threshold.type === 'critical') {
    // Terminate non-essential instances
    const nonEssential = await db.query(`
      SELECT id FROM instances
      WHERE provider = 'aws'
        AND workflow_state LIKE '%:complete'  -- Warm pool instances (availability inferred)
        AND current_manifest_id IS NULL
    `);

    for (const instance of nonEssential) {
      await submitWorkflow({
        type: 'terminate-instance',
        input: { instanceId: instance.id, reason: 'budget_limit' },
      });
    }
  }
```

### TTL Patterns

```typescript
/**
 * TTL patterns for provider state management.
 */

interface TTLConfig {
  /** Default TTL for uncategorized entries */
  default: number;

  /** TTL by key prefix */
  byPrefix: Record<string, number>;

  /** Maximum allowed TTL */
  maxTtl: number;
}

const AWS_TTL_CONFIG: TTLConfig = {
  default: 300_000,  // 5 minutes
  maxTtl: 3600_000,  // 1 hour

  byPrefix: {
    'spot_prices:': 300_000,       // 5 minutes
    'capacity:': 60_000,           // 1 minute
    'instance:': 30_000,           // 30 seconds
    'warm_volume_pool:': null,     // No TTL (explicitly managed)
    'cost:': 3600_000,             // 1 hour
    'api_calls:': 60_000,          // 1 minute (rolling window)
  },
};

/**
 * Get appropriate TTL for a key.
 */
function getTTL(provider: ProviderName, key: string): number | null {
  const config = PROVIDER_TTL_CONFIGS[provider] ?? { default: 300_000, byPrefix: {}, maxTtl: 3600_000 };

  for (const [prefix, ttl] of Object.entries(config.byPrefix)) {
    if (key.startsWith(prefix)) {
      return ttl;
    }
  }

  return config.default;
}

/**
 * Wrapper that automatically applies TTL.
 */
async function setWithAutoTTL<T>(
  provider: ProviderName,
  key: string,
  value: T
): Promise<void> {
  const ttl = getTTL(provider, key);
  await providerState.set(provider, key, value, ttl ?? undefined);
}
```

### Hooks vs Workflows Distinction

```
HOOKS                                  WORKFLOWS
----------------------------------------------------------------------
Event-driven                           Task-driven
Synchronous response expected          Asynchronous execution
Lightweight (< 1 second)               May be long-running (minutes)
Cannot spawn subworkflows              Can spawn subworkflows
No manifest/resource ownership         Owns manifest with resources
Called by control plane lifecycle      Submitted via workflow engine
Cannot use pattern library             Uses pattern library
Returns data to caller                 Writes output to workflow record

WHEN TO USE HOOKS:
- Quick state updates (onInstanceHeartbeat)
- Cache warming (onStartup)
- Triggering workflows (onSpotInterruption -> submit workflow)
- Lightweight maintenance (onHeartbeat)

WHEN TO USE WORKFLOWS:
- Multi-step operations (spawn, terminate)
- Operations that create resources
- Operations that need compensation capability
- Operations with timeout/retry requirements
```

```typescript
/**
 * Hook triggers workflow example.
 * Hooks are the bridge between events and workflows.
 */

// This is a HOOK - responds to event, triggers workflow
async function onSpotInterruption(notice: SpotInterruptionNotice): Promise<void> {

      warningTimeMs: notice.warningTimeMs,

  });
}

      handler: 'spawnReplacementInstance',

    },
    'resume-run': {
      handler: 'resumeRunFromCheckpoint',
      next: 'finalize',
    },
    'finalize': {
      handler: 'finalizeInterruptionHandling',
      next: null,
    },
  },
};
```

### Provider Registration

```typescript
/**
 * Complete provider registration including hooks and workflows.
 */

interface ProviderRegistration {
  /** Provider implementation */
  provider: Provider;

  /** Lifecycle hooks */
  hooks: ProviderLifecycleHooks;

  /** Provider-specific workflows */
  workflows: WorkflowBlueprint[];

  /** Workflow contracts */
  contracts: ProviderWorkflowContract[];
}

/**
 * Register a provider with the control plane.
 */
function registerProvider(registration: ProviderRegistration): void {
  const { provider, hooks, workflows, contracts } = registration;

  // Register provider instance
  providers.set(provider.name, provider);

  // Register hooks
  if (hooks) {
    providerHooks.set(provider.name, hooks);
  }

  // Register workflows and contracts
  for (const workflow of workflows) {
    workflowRegistry.register(workflow);
  }

  for (const contract of contracts) {
    providerWorkflowContracts.set(`${provider.name}:${contract.type}`, contract);
  }

  debug('PROVIDER/register', {
    provider: provider.name,
    capabilities: provider.capabilities,
    workflowCount: workflows.length,
    hasHooks: !!hooks,
  });
}

// Example: Full AWS registration
registerProvider({
  provider: new AWSProvider(),
  hooks: awsLifecycleHooks,
  workflows: [
    AWS_WARM_VOLUME_POOL_WORKFLOW,
    AWS_SPAWN_WITH_WARM_VOLUME_WORKFLOW,
    AWS_SPOT_FLEET_WORKFLOW,
  ],
  contracts: [
    AWS_WARM_VOLUME_POOL_CONTRACT,
    AWS_SPAWN_WITH_WARM_VOLUME_CONTRACT,
    AWS_SPOT_FLEET_CONTRACT,
  ],
});

// Example: Simple Lambda registration (Tier 2)
registerProvider({
  provider: new LambdaProvider(),
  hooks: {
    onStartup: async () => {
      await verifyLambdaApiKey();
    },
    onHeartbeat: async (expectations) => {
      return await handleLambdaHeartbeat(expectations);
    },
  },
  workflows: [],  // No custom workflows
  contracts: [],  // No custom contracts
});
```

### Contract Testing Pattern (Vitest)

Contract compliance is verified through parametric tests that run the same
test suite against all providers. This uses a registry pattern for both
runtime and test-time provider discovery.

```typescript
/**
 * Provider registry for runtime and test discovery.
 * Shared between production code and test suite.
 *
 * Providers are organized into:
 * - compute/: Cloud compute providers (AWS, Lambda Labs, RunPod, OrbStack)
 * - feature/: Cross-cutting feature providers (Tailscale, etc.)
 */

// control/src/provider/registry.ts
export const providerRegistry = {
  aws: () => import('./compute/aws'),
  lambda: () => import('./compute/lambda'),
  runpod: () => import('./compute/runpod'),
  orbstack: () => import('./compute/orbstack'),
} as const;

export type ProviderName = keyof typeof providerRegistry;

/**
 * Helper to get provider by name (runtime).
 */
export async function getProvider(name: ProviderName): Promise<Provider> {
  const mod = await providerRegistry[name]();
  return mod.default;
}
```

```typescript
/**
 * Contract compliance tests.
 * Run same suite against all providers using describe.each.
 */

// tests/control/unit/provider-contracts.test.ts
import { describe, it, expect, beforeAll } from 'vitest';
import { providerRegistry, ProviderName } from '../../../control/src/provider/registry';
import type { Provider } from '../../../control/src/provider/types';

const providers = Object.keys(providerRegistry) as ProviderName[];

describe.each(providers)('Provider %s', (name) => {
  let provider: Provider;

  beforeAll(async () => {
    const mod = await providerRegistry[name]();
    provider = mod.default;
  });

  describe('SpawnInstanceContract compliance', () => {
    it('returns required fields on success', async () => {
      const result = await provider.spawn(testOptions);
      expect(result).toHaveProperty('id');
      expect(result).toHaveProperty('status');
      expect(result.status).toMatch(/pending|starting|running/);
    });

    it('is idempotent when capability declared', async () => {
      if (!provider.capabilities.idempotentSpawn) {
        // Skip test for providers without idempotent spawn
        return;
      }

      const opts = { ...testOptions, clientRequestId: 'idem-1' };
      const r1 = await provider.spawn(opts);
      const r2 = await provider.spawn(opts);
      expect(r1.id).toBe(r2.id);
    });

    it('emits required tags', async () => {
      const result = await provider.spawn({
        ...testOptions,
        tags: { 'skyrepl:test': 'true' },
      });
      const instance = await provider.get(result.id);
      expect(instance?.metadata?.tags).toHaveProperty('skyrepl:test', 'true');
    });
  });

  describe('TerminateInstanceContract compliance', () => {
    it('handles non-existent instance gracefully', async () => {
      await expect(provider.terminate('nonexistent'))
        .rejects.toThrow(NotFoundError);
    });

    it('is idempotent (terminate twice = same result)', async () => {
      const instance = await provider.spawn(testOptions);
      await provider.terminate(instance.id);

      // Second terminate should not throw
      await expect(provider.terminate(instance.id))
        .resolves.not.toThrow();
    });
  });

  describe('ListFilter compliance', () => {
    it('filters by region', async () => {
      const results = await provider.list({ region: 'us-east-1' });
      expect(results.every(i => i.region?.startsWith('us-east-1'))).toBe(true);
    });

    it('filters by status', async () => {
      const results = await provider.list({ status: ['running', 'pending'] });
      expect(results.every(i => ['running', 'pending'].includes(i.status))).toBe(true);
    });
  });

  describe('BootstrapScript compliance', () => {
    it('generates valid bootstrap script', async () => {
      const script = provider.generateBootstrap(testBootstrapConfig);
      expect(script.content).toBeTruthy();
      expect(script.format).toMatch(/cloud-init|shell|ignition/);
      expect(script.checksum).toHaveLength(64); // SHA-256
    });

    it('includes agent download in bootstrap', async () => {
      const script = provider.generateBootstrap(testBootstrapConfig);
      expect(script.content).toContain(testBootstrapConfig.agentUrl);
    });
  });

  // Capability-gated tests
  describe('Snapshot capability', () => {
    it('creates snapshot when supported', async () => {
      if (!provider.capabilities.snapshots) return;

      const instance = await provider.spawn(testOptions);
      const request = await provider.createSnapshot!(instance.id, {
        name: 'test-snapshot',
      });

      expect(request).toHaveProperty('requestId');
      expect(request.status.status).toMatch(/pending|creating/);
    });
  });

  describe('Spot capability', () => {
    it('requests spot instance when supported', async () => {
      if (!provider.capabilities.spot) return;

      const request = await provider.requestSpotInstance!({
        ...testOptions,
        maxSpotPrice: 0.50,
      });

      expect(request).toHaveProperty('requestId');
      expect(request.status).toMatch(/pending|active|fulfilled/);
    });
  });
});
```

This pattern ensures:
- Registry serves both runtime and test-time provider discovery
- Parametric tests run same suite against all providers
- Capability-gated tests skip gracefully for providers without features
- Contract compliance is explicit, not buried in integration tests
- Providers implementing contracts inherit validated behavior

### Summary: Provider Implementation Spectrum

```
TESTING & CHAOS PROVIDER (OrbStack)
-----------------------------------
OrbStack is the foundation of contract-based testing.

Intent contracts (SpawnInstanceContract, TerminateInstanceContract, etc.) define
provider-agnostic behavior. OrbStack implements these contracts identically to
cloud providers, plus chaos injection for failure scenarios.

Key insight: We're not testing "OrbStack works therefore AWS works." We're testing
"the control plane handles intent contracts correctly, and any provider implementing
those contracts gets the same behavior."

Chaos capabilities:
- Fault injection (network timeouts, API errors, spot interruptions)
- Resource constraints (capacity errors, quota limits)
- Drift simulation (instances disappearing, IPs changing)
- Contract compliance verification under failure conditions

Requirements: Provider interface core methods, chaos/fault injection hooks,
controlled failure scenarios, error taxonomy mapping.

Example Implementation Size: ~600 lines (including chaos features)


ADDITIONAL PROVIDERS (Lambda, RunPod, future: SLURM, GCP, CoreWeave)
---------------------------------------------------------------------
Requirements:
- Implement Provider interface core methods
- Implement basic lifecycle hooks (onStartup, onShutdown, onHeartbeat)
- Handle 'health_check' and 'reconcile' heartbeat task types
- Map provider errors to error taxonomy

Optional:
- Implement optional interface methods (snapshots, spot) based on capabilities
- Use in-memory caching

Not Needed:
- Custom workflows
- Custom workflow contracts
- Complex hooks (onSpotInterruption, etc.)

Example Implementation Size: ~500 lines

Note: Simpler providers aren't forced into complex cloud platform integration.
Each provider implements the capabilities that make sense for its platform.


REFERENCE IMPLEMENTATION (AWS - first detailed integration)
------------------------------------------------------------
Requirements:
- Everything from Additional Providers

Plus:
- Enhanced types (AWSInstance, AWSSnapshot)
- Materialization/projection functions
- Custom workflows for optimization:
  - Warm volume pool management
  - Spot fleet orchestration
  - Multi-AZ capacity probing
- Full lifecycle hooks:
  - Full heartbeat expectations handling (all task types)
  - Workflow launch reporting (spot interruptions, budget thresholds)
  - Sophisticated reconciliation with drift detection
- Provider-specific workflow contracts

Advanced:
- Cost Explorer integration
- Capacity Advisor integration
- Reserved instance utilization tracking

Example Implementation Size: ~3000 lines

AWS happened to be the first provider with deep integration. When SLURM,
GCP, or CoreWeave are added, they will receive the same depth of integration
where appropriate. AWS is not inherently superior - it's just the reference
implementation that established patterns for advanced provider features.


The key difference: Advanced providers use the extension framework to
implement optimizations that are invisible to the control plane. The
control plane submits workflows and receives generic outputs. All
provider-specific complexity is encapsulated.
```
