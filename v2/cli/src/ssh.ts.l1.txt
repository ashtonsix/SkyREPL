# ssh.ts - SSH Configuration

## Config Generation
- Write to ~/.repl/ssh_config
- Per-allocation routing (not per-instance)

## Host Entry Format
- Host: allocation ID (e.g., alloc-abc123)
- HostName: instance IP
- User: allocation user
- RemoteCommand: cd to workdir

## Tailscale Integration
- ProxyCommand via ssh_proxy_command.sh
- JIT Tailscale connection

## Triggers
- Auto-regenerate on allocation state changes
- Update on IP changes

## Shared Instance Notice
Before connecting, check if target instance has other active allocations:
- Query GET /v1/instances/:id/allocations
- If other ACTIVE allocations exist, display notice:
  ```
  Notice: Instance has N other active allocation(s)
    alloc-xyz (run-789) - ACTIVE, started 5m ago
  Resources (CPU, memory, disk I/O) are shared.
  ```
- Notice is informational (non-blocking)
- Helps users understand why debugging may affect/be affected by other work
