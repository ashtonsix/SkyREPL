# integration/api-resources.test.ts - Layer 1 API integration tests

## SETUP
- Start test control plane on random port
- Clean database before each test
- Seed test data where needed
- HTTP client configured with base URL
- Shutdown control plane after suite

## Runs CRUD
- GET /api/runs list with pagination
  - Returns array of runs
  - Pagination: limit=10, offset=0 default
  - Total count in response header
  - Links to next/prev page
  - Sorted by created_at desc
- GET /api/runs/:id single
  - Returns run object
  - Includes basic fields
  - 404 if not found
  - ID validation (must be valid format)
- POST /api/runs create
  - Creates new run
  - Returns 201 with Location header
  - Command is required
  - Spec_checksum is required
  - Optional: env, files, init_checksum
- PUT /api/runs/:id update allowed fields
  - Can update status
  - Can update workflow_state
  - Can update exit_code
  - Cannot update created_at
  - Cannot update ID
  - Returns updated run
- DELETE /api/runs/:id if not running
  - Deletes run in PENDING status
  - Deletes run in COMPLETE status
  - 409 error if status=RUNNING
  - Cascades to allocations (if applicable)

## Instances CRUD
- GET /api/instances list
  - Returns array of instances
  - Filter by provider: ?provider=orbstack
  - Filter by status: ?status=running
  - Pagination works
  - Include allocations: ?include=allocations
- GET /api/instances/:id single
  - Returns instance object
  - Provider metadata included
  - 404 if not found
- POST /api/instances create
  - Creates new instance record
  - provider and provider_id required
  - spec_checksum required
  - Status defaults to 'pending'
  - Returns 201
- PUT /api/instances/:id update
  - Can update status
  - Can update provider_metadata
  - Cannot update provider (immutable)
  - Cannot update provider_id (immutable)
- DELETE /api/instances/:id
  - Deletes instance
  - 409 if has active allocations
  - Cascades to COMPLETE allocations
- GET /api/instances/:id with include query
  - ?include=allocations returns allocations array
  - ?include=runs returns associated runs
  - Multiple includes: ?include=allocations,runs
- POST /api/instances/:id/refresh_external
  - Queries provider API for current state
  - Updates instance record
  - Returns updated instance
  - 404 if provider not found

## Allocations CRUD
- GET /api/allocations list
  - Returns array of allocations
  - Filter by instance_id
  - Filter by run_id
  - Filter by status
  - Pagination works
- GET /api/allocations/:id single
  - Returns allocation object
  - Includes instance reference
  - Includes run reference (if claimed)
  - 404 if not found
- POST /api/allocations create
  - instance_id required
  - status defaults to AVAILABLE
  - init_checksum optional
  - workdir auto-generated
  - Returns 201
- PUT /api/allocations/:id update
  - Can update status
  - Can update run_id (when claiming)
  - Can update hold_until (extend hold)
  - Cannot update instance_id
  - Cannot update workdir
- DELETE /api/allocations/:id
  - Deletes allocation
  - 409 if status=ACTIVE
  - Allowed if AVAILABLE or COMPLETE
- PUT /api/allocations/:id hold extension
  - Updates hold_until timestamp
  - Validates max hold duration
  - 400 if exceeds max
  - Returns updated allocation
- Status transitions validated
  - AVAILABLE→CLAIMED allowed
  - CLAIMED→ACTIVE allowed
  - ACTIVE→HELD allowed
  - HELD→COMPLETE allowed
  - Invalid transitions → 400 error

## Workflows Read-Only
- GET /api/workflows list
  - Returns array of workflows
  - Filter by status
  - Filter by manifest_id
  - Pagination works
  - Does not include nodes by default
- GET /api/workflows/:id single
  - Returns workflow object
  - Includes input/output
  - ?include=nodes returns nodes array
  - Nodes include dependencies
  - 404 if not found
- No POST endpoint
  - POST /api/workflows → 404
  - Use Layer 2 API for workflow creation
  - Error message guides to correct endpoint

## Error Responses
- 400 validation errors
  - Missing required field
  - Invalid field type
  - Invalid enum value
  - Consistent error envelope: {error, message, details}
- 404 not found
  - Invalid resource ID
  - Resource deleted
  - Message includes resource type
- 409 conflicts
  - Constraint violations
  - Invalid state transitions
  - Message explains conflict
- Error envelope consistency
  - All errors use same format
  - HTTP status matches error type
  - Details array for validation errors
