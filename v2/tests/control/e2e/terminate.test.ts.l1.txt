# e2e/terminate.test.ts - Terminate E2E tests (OrbStack)

## SETUP
- OrbStack provider configured
- Control plane running
- CLI available
- Test instances created
- Clean database and OrbStack between tests

## Normal Terminate Tests
- Instance terminated via workflow
  - repl terminate <instance_id> command
  - Termination workflow created
  - Returns workflow_id
  - Streams progress
- Allocations cleaned up
  - All allocations for instance updated
  - ACTIVE allocations transitioned to COMPLETE
  - AVAILABLE allocations removed
  - COMPLETE allocations remain (history preserved)
  - Debug holds checked before termination
- Tailscale deregistered
  - Instance removed from Tailscale network
  - Tailscale API called
  - Node deleted from network
  - Verified via Tailscale CLI
- Records updated
  - Instance status → terminated
  - terminated_at timestamp set
  - Provider metadata updated
  - Manifest sealed

## Force Terminate Tests
- Skip SSH drain with --force flag
  - repl terminate --force <instance_id>
  - Drain step skipped
  - Active SSH sessions killed
  - Immediate shutdown
- Immediate termination
  - No waiting for graceful shutdown
  - Provider API called immediately
  - Instance deleted
  - Faster completion (seconds vs minutes)

## With Active SSH Tests
- Drain timeout respected
  - Active SSH session present
  - Drain timeout configured (e.g., 5 minutes)
  - Waits for SSH to disconnect
  - Terminates after timeout if still connected
- SSH users notified
  - Warning message sent to SSH session
  - MOTD updated with termination warning
  - Countdown displayed (if possible)
  - Connection closed gracefully

## Provider Cleanup Tests
- Instance deleted from provider
  - OrbStack container stopped
  - Container removed
  - Volumes deleted
  - Network cleaned up
- No orphans remain
  - Query OrbStack for containers
  - Verify instance not in list
  - No dangling volumes
  - No dangling networks
  - Reconciliation check passes
