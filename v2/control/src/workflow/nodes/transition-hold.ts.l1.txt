# transition-hold.ts - DEPRECATED

## Status: DEPRECATED

This node has been removed. Its functionality has been integrated into the finalize node.

## Previous Behavior

Previously, this node transitioned allocations from ACTIVE → HELD.

## Current Behavior

The finalize node now handles:
- ACTIVE → COMPLETE transition
- Setting debug_hold_until (optional, for debugging)
- Warm pool replenishment (creating new AVAILABLE allocation)
- Manifest sealing
- Optional snapshot creation

## Migration Notes

If you see references to transition-hold in workflow definitions:
- Replace with finalize node
- Configure debug_hold_until in finalize node input if debugging needed
- Warm pool replenishment happens automatically in finalize
