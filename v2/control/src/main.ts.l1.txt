# control.ts - Control Plane

## Startup
- `main(): void` - Entry point
  - Load config from environment
  - Initialize database, run migrations
  - Start workflow engine
  - Register providers and workflow blueprints
  - Start HTTP server (ElysiaJS)
  - Start background tasks
  - Register signal handlers (SIGINT, SIGTERM)

## Database
- `initDatabase(): Database` - Open SQLite at ~/.repl/control.db
- `runMigrations(): void` - Apply schema migrations

## Workflow Engine
- `startWorkflowEngine(): WorkflowEngine`
  - Register node executors (spawn-instance, sync-files, etc.)
  - Register workflow blueprints (launch-run, terminate, snapshot, etc.)
  - Start background worker

## Providers
- `registerProviders(): void` - Load AWS, Lambda, RunPod, OrbStack
- `registerProviderHooks(): void` - Register spot_terminated hooks

## HTTP Server (ElysiaJS)
- `startHttpServer(): void` - Listen on port 8080
- Mount routes:
  - Layer 1: `/v1/runs`, `/v1/instances`, `/v1/allocations`, `/v1/workflows`, `/v1/manifests`, `/v1/objects`
  - Layer 2: `/v1/workflows/launch-run`, `/v1/workflows/terminate-instance`, etc.
  - Agent: `/v1/agent/heartbeat`, `/v1/agent/logs`, `/v1/agent/commands`
- WebSocket handlers: `/v1/runs/:id/logs`, `/v1/workflows/:id/progress`

## Background Tasks (periodic)
- `heartbeatTimeoutCheck()` - Every 30s, detect stale heartbeats
- `holdExpiryCheck()` - Every 1m, release expired holds
- `manifestCleanupCheck()` - Every 5m, cleanup expired manifests
- `reconciliationTask()` - Every 60s, sync provider state (see below)
- `storageGarbageCollection()` - Every 24h, delete unreferenced blobs

### reconciliationTask() Implementation
- For each enabled provider:
  - Call provider.list({ namePrefix: 'repl-', includeTerminated: false })
  - Query db.getAllInstances({ provider }) to get tracked instances
  - Cross-reference: Find orphans (in provider but not in DB)
  - Cross-reference: Find missing (in DB but not in provider)
  - If orphans found:
    - Log orphan detection: "Found {count} orphan(s) in {provider}"
    - Check if cleanup workflow is enabled via config.enableAutomaticOrphanCleanup
    - If enabled: Submit cleanup workflow with { provider, orphans, autoTerminate: false }
    - If disabled: Log to audit trail only (no workflow submission)
  - If missing resources found:
    - Log missing resources for operator review
    - Update DB records to mark as 'lost' or 'provider_terminated'
- Track session start time for orphan classification (current vs. other sessions)
- Emit metrics: orphan_count, missing_count, provider_sync_duration_ms

## Shutdown
- `shutdown(): void` - Wait for running workflows, close database, exit
