# spot.py - Spot Monitor (only interruptible instances)

# Note: Timing constants reference ../../shared/src/config/timing.ts.l2.txt
# - SPOT_CHECK_INTERVAL_MS (5s)
# - SPOT_CHECKPOINT_BUDGET_MS (90s)

## Loop
Every SPOT_CHECK_INTERVAL_MS (5s): Check EC2 metadata for interruption notice

## IMDSv2
GET token, then GET /latest/meta-data/spot/instance-action

## On Interruption (2 min warning)
1. Send SpotInterruptStartMessage (POST /v1/agent/spot-interrupt-start)
   - Include time_remaining_ms (from IMDS response)
   - Signals: Interrupt detected, checkpoint starting
2. Execute checkpoint via shared logic (checkpoint.py):
   - Call execute_spot_checkpoint(run_id, artifact_urls)
   - Uses SPOT_CHECKPOINT_BUDGET_MS (90s) timeout
   - Set SKYREPL_CHECKPOINT_REASON=spot_termination
   - Handles artifact upload with failover URLs
3. Send SpotInterruptCompleteMessage (POST /v1/agent/spot-interrupt-complete)
   - Include checkpoint_exit_code (from checkpoint result)
   - Include artifacts_uploaded (count from checkpoint result)
   - Signals: Checkpoint done, ready for termination

## Two-Phase Protocol
- Phase 1: SpotInterruptStartMessage signals interrupt detected
- Phase 2: SpotInterruptCompleteMessage signals checkpoint and upload complete
- Control waits for spot_interrupt_complete before transitioning to FAILED
- Timeout: 30s from spot_interrupt_start
- If timeout expires: Control transitions to FAILED anyway (artifact loss acceptable)

## Timing Budget (120s total)
- Checkpoint: SPOT_CHECKPOINT_BUDGET_MS (90s max)
- Artifact upload: 20s
- Messaging overhead: 10s buffer

## Artifact Prioritization
Spot uses **size ascending** strategy to maximize artifact count in tight 20s window.
- Smallest files uploaded first
- If budget exhausted, larger files are skipped
- Rationale: 10 small checkpoints > 1 large checkpoint when time is critical
- See checkpoint.py for prioritization implementation

## Shared Logic
Checkpoint execution shared with heartbeat panic (see checkpoint.py.l2.txt).
Both paths use execute_checkpoint() with different timeout budgets.
Different prioritization strategies: spot uses size_asc, panic uses recency_desc.
