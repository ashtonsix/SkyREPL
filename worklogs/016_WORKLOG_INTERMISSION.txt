STEP 17: Bugs surfaced by E2E testing .................................. [x] DONE
  a. create-allocation.ts: warm path node ID "claim-allocation" →
     "claim-warm-allocation" (warm path was never tested E2E)
  b. agent.ts: closeRunSubscribers() never called on terminal status.
     CLI WebSocket log streams never received terminal message. Added
     closeRunSubscribers() call in /v1/agent/status handler.
  c. executor.py: flush_all() before _send_status() — log buffer
     flushed on timer, but status POST was immediate, so control plane
     closed CLI WebSockets before logs were delivered.
  d. agent.ts waitForEvent: polling interval 1000ms → 200ms. Was the
     dominant cost in simulated E2E (31s → 1.6s for 7 tests).

STEP 18: OrbStack bootstrap fix ......................................... [x] DONE
  Root cause: Single-command `nohup /tmp/bootstrap.sh ... &` via orbctl
  run was unreliable — backgrounded process killed when orbctl exits.
  Fix: Two-step bootstrap (write script, then start with setsid + sleep).
    Step 1: orbctl run → write /tmp/bootstrap.sh
    Step 2: orbctl run → setsid /tmp/bootstrap.sh ... & sleep 0.5
