Checksum Staleness Design
-------------------------

For appendable objects (e.g., logs):
- `checksum_bytes` tracks how many bytes the checksum covers
- When `checksum_bytes == size_bytes`: checksum is current
- When `checksum_bytes < size_bytes`: checksum is stale (needs recomputation)
- NULL when checksum is NULL

Reservation Field Design
------------------------

For objects expected to grow beyond 4KB:
- Set `reservation` to expected final size
- Invariant: `reservation >= size_bytes` when set
- If `reservation > 4KB`, object goes directly to S3 (skips inline storage)
- Use cases: logs, streaming uploads, files known to be large
