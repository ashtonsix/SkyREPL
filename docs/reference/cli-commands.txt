# CLI Command Reference

Extracted from spec/10_CLI.txt §10.3-10.8

## 10.3 Run Commands

### repl run [profile]

Submit a launch-run workflow.

**Usage:**
```bash
repl run                           # Use default profile
repl run dev                       # Use named profile
repl run --command "python x.py"   # Override command
repl run --timeout 2h --spot       # Override settings
repl run --dry-run                 # Validate without launching
```

**Flags:**
```
--command CMD       Override profile command
--timeout DURATION  Override max duration (10m, 2h, 30s)
--workdir PATH      Override remote workdir
--spot, --no-spot   Force spot/on-demand
--hold DURATION     Keep allocation after run (default: 5m)
--snapshot          Create snapshot on success
--dry-run           Validate config and files, don't launch
--no-stream         Submit and exit (don't stream progress)
```

**Output (TTY):**

Progress display shows workflow and run IDs, node status (starting/done/failed),
progress bars for file sync, run output once execution begins, and a summary
receipt on completion (run ID, instance, duration, exit code, cost, allocation
info, SSH shortcut, artifact pull command). Exact formatting emerges from
implementation; key principle is workflow progress via SSE, then actionable
next steps.

**Output (JSON, --no-stream):**
```json
{
  "workflow_id": "wf-abc123",
  "run_id": "run-xyz789",
  "status": "pending",
  "stream_url": "/v1/workflows/wf-abc123/stream"
}
```

### repl run status <run-id>

Get run status (distinct from `repl status` which shows control plane health).

**Usage:**
```bash
repl run status run-xyz789   # Specific run status
repl run status run-xyz      # Prefix resolution
```

> See also: impl/cli/src/commands/run.ts

**Output:** Run ID, status, workflow state, instance info, allocation, timestamps, duration.

### repl logs <run-id>

Stream logs from a run.

**Usage:**
```bash
repl logs run-xyz789         # Stream logs (follows)
repl logs run-xyz789 --tail  # Last 100 lines, then follow
repl logs run-xyz789 --head  # First 100 lines, exit
```

**Behavior:**
- Connects to WebSocket at `/v1/runs/:id/logs`
- Streams stdout/stderr to terminal
- Ctrl-C disconnects from logs (run continues)

### repl list

List recent runs.

**Usage:**
```bash
repl list                    # Last 50 runs (default)
repl list --limit 100        # More runs
repl list --status running   # Filter by status
repl list --all              # Include archived
```

**Output:**
```
ID          STATUS     DURATION   COST     CREATED
run-xyz789  running    3m 42s     $0.12    2024-01-31 14:32
run-abc123  completed  45m 12s    $1.23    2024-01-31 13:00
run-def456  failed     2m 05s     $0.08    2024-01-31 12:15
```

Defaults to 50 results per page (matching API default), adjustable with `--limit`.

### repl cancel <run-id>

Cancel a run and its workflow.

**Usage:**
```bash
repl cancel run-xyz789       # Cancel run
repl cancel run-xyz          # Prefix resolution
repl cancel --force run-xyz  # Skip compensation
```

**Output:**
```
Cancelling run run-xyz789...
Workflow wf-abc123: rolling_back
  [sync-files] rolled back
  [claim-allocation] released
Run cancelled.
```

### repl extend <allocation-id>

Extend allocation hold period.

**Usage:**
```bash
repl extend alloc-def456              # Extend by default (30m)
repl extend alloc-def456 --duration 2h
repl extend alloc-def                 # Prefix resolution
```

**Output:**
```
Extended allocation alloc-def456
Previous hold: 14:50:32
New hold:      15:20:32 (+30m)
Max hold:      2024-02-01 14:32:02 (24h from start)
```

> See also: impl-pseudo/cli/src/ssh.ts.l2.txt §CliCommandReplExtend

## 10.4 Resource Commands

### repl artifact list/pull

**Usage:**
```bash
repl artifact list                      # All artifacts
repl artifact list --run run-xyz789     # Specific run
repl artifact pull run-xyz789           # Download all
repl artifact pull run-xyz789 --dest ./results
repl artifact pull run-xyz789 --pattern "*.pt"
```

**List Output:** Table showing run ID, artifact path, size, and creation time.

**Pull Output:** Progress display with download progress, then confirmation of destination.

### repl instance list/terminate

**Usage:**
```bash
repl instance list                      # All instances
repl instance terminate inst-abc123     # Terminate
repl instance terminate inst-abc --yes  # Skip confirmation
```

**List Output:** Table with instance ID, provider, spec, status, IP, and allocation count.

### repl snapshot list

List available snapshots.

**Usage:**
```bash
repl snapshot list                      # All snapshots
repl snapshot list --provider aws       # Filter by provider
repl snapshot list --run run-xyz789     # Snapshots from specific run
```

**Output:** Table showing snapshot ID, provider, source run, size, created timestamp,
and status (creating, available, failed).

### repl orphan list/whitelist

**Usage:**
```bash
repl orphan list                        # List all orphaned resources
repl orphan list --provider aws         # Filter by provider
repl orphan list --type instance        # Filter by resource type
repl orphan whitelist <resource-id>     # Mark as intentionally unmanaged
```

Lists cloud resources not tracked in the database. See Chapter 12 (Orphans) for
detection algorithm and whitelist semantics.

**Output:** Table with resource ID, provider, type, created timestamp, estimated daily cost.

### repl usage [period]

Usage and cost reporting.

**Usage:**
```bash
repl usage                              # Current month
repl usage last-month                   # Previous month
repl usage 2024-01                      # Specific month
repl usage --provider aws               # Filter by provider
repl usage --export usage.csv           # Export to CSV
```

**Output:** Summary showing:
- Total cost and hours by provider
- Cost breakdown by instance type
- Run count and success rate
- Spot vs on-demand utilization
- Cost trends (if period covers multiple months)

## 10.5 Setup and Initialization

### repl init

Interactive setup wizard for first-time users.

**Usage:**
```bash
repl init                   # Start interactive setup
repl init --noninteractive  # Use defaults where possible
```

**Setup Sequence:**

1. **Provider Setup** - Select providers (AWS, Lambda Labs, RunPod), configure
   each via `repl setup-provider`, verify connectivity

2. **Control Plane Setup** - Choose local or cloud deployment mode:
   - Local: configure MinIO, data directory; optionally enable Funnel to expose
     control plane so remote agents can connect without joining Tailnet
   - Cloud: configure endpoint and authentication
   - Initialize database schema, start and verify health

3. **Local Environment Setup** - Generate SSH config include snippet with
   instructions, create ~/.repl directory structure, generate example repl.toml

4. **Verification** - Run `repl verify-setup`, display summary, show example
   `repl run` command

**Output:** Progress per step; clear errors with resolution suggestions; welcome
message with quick start examples on success.

## 10.6 Control Plane Commands

### repl control start/stop/status

**Usage:**
```bash
repl control start                # Start control plane
repl control start --funnel       # With Funnel tunnel
repl control stop                 # Stop control plane
repl control status               # Health check
```

**Start Output:** Shows startup progress for MinIO, Funnel (if enabled), and control plane.
Reports URLs and log file location when ready.

**Status Output:** Health check showing:
- API status and version
- Database connectivity and basic stats
- S3/storage status
- Tailscale status
- Provider connectivity per configured provider

### repl setup-provider <provider>

Interactive provider setup.

**Usage:**
```bash
repl setup-provider aws
repl setup-provider lambda
repl setup-provider runpod
```

**Behavior:**

Interactive wizard that collects:
1. Provider credentials (API keys, access tokens)
2. Region selection
3. SSH key configuration
4. Connectivity verification

Saves configuration to `~/.repl/control.env`. Prompt format emerges from implementation.

### repl verify-setup

Verify all provider connectivity.

**Usage:**
```bash
repl verify-setup
repl verify-setup --provider aws
```

**Output:**

Per-provider verification showing:
- Credential validity
- Region/availability status
- SSH key status
- Required permissions (provider-specific)

Summary with overall pass/fail.

## 10.7 Authentication

### Token-Based Authentication

**Token Storage:** `~/.repl/auth.json` with format `{"control_plane_url": "token_value"}`,
file permissions 0600.

**Token Lifecycle:** Issued during `repl init` or `repl auth login`; includes
expiry timestamp checked before each API call; automatic refresh if refresh
token available; re-authentication if refresh fails.

For multi-user authentication, see Chapter 15.7 Security.

### repl auth login/logout/status

**Usage:**
```bash
repl auth login                         # Interactive login
repl auth login --token TOKEN           # Use existing token
repl auth logout                        # Clear authentication
repl auth status                        # Check auth status
```

**Login Behavior:**

For local control plane:
- Auto-generate and store token (no user input needed)
- Token tied to machine ID

For cloud control plane (funnel):
- Prompt for email/password or OAuth flow
- Store token and refresh token
- Display token expiry

**Status Output:**
```
Authenticated with control plane at https://localhost:8080
Token expires: 2024-03-01 14:32:00 (27 days remaining)
User: local-dev
```

**Logout Behavior:**
Removes token from `~/.repl/auth.json` without server-side revocation (tokens have TTL).

## 10.8 Updates and Feedback

### repl update

Check for and apply CLI updates.

**Usage:**
```bash
repl update                             # Check and prompt to update
repl update --check                     # Check only, don't update
repl update --yes                       # Auto-apply without prompt
```

**Behavior:** Query update server, compare versions (semver), show changelog if
update available, prompt to update (unless --yes), download and replace binary,
verify installation.

**Update Check Frequency:** Every 7 days (configurable), timestamp in
`~/.repl/state.json`, notification on next command if available, never
interrupts running workflows.

**Output:**
```
Current version: v1.2.3
Latest version:  v1.3.0

Changelog:
- Added snapshot list command
- Fixed issue with long-running SSH connections
- Improved error messages for provider failures

Update now? [Y/n]
```

### repl feedback

Submit feedback or bug reports.

**Usage:**
```bash
repl feedback                           # Interactive feedback form
repl feedback --bug                     # Bug report template
repl feedback --feature                 # Feature request template
```

Opens interactive form or editor ($EDITOR). Template includes type
(bug/feature/general), summary, details, reproduction steps (if bug),
environment info, optional log attachment (user consent), optional contact email.

**Submission:** Posts to feedback API, assigns ticket ID, confirms with ID.
Optional diagnostic bundle (sanitized logs, config).

**Privacy:** Sanitizes API keys/tokens from logs; explicit opt-in for log
attachment; shows data before submission.
