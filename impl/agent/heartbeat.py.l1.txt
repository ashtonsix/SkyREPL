# heartbeat.py - Heartbeat Thread

# Note: Timing constants reference ../../shared/src/config/timing.ts.l2.txt
# - HEARTBEAT_INTERVAL_MS (10s)
# - HEARTBEAT_DEGRADED_MS (2m) - early warning threshold
# - HEARTBEAT_ACK_TIMEOUT_MS (15m) - agent self-termination trigger
# - SSH_ACTIVITY_SLACK_MS (60s)

## Critical
Send IMMEDIATE heartbeat before first sleep (closes timing gap)
Store control_plane_id from first ack (None until then)

## Loop
Every HEARTBEAT_INTERVAL_MS (10s): POST /v1/agent/heartbeat
Payload: tailscale_ip, active_ssh_sessions, tailscale_status, workflow_state, pending_command_acks
Response: ack, control_plane_id (validate ID matches expected)

## Degraded Detection (Early Warning)
At HEARTBEAT_DEGRADED_MS (2m) without ack:
- Set workflow_state = 'degraded:control_plane_unreachable'
- Continue heartbeating (control plane may recover)
- Provides UX visibility before the 15m cut

## Panic Checkpoint (Self-Termination)
At HEARTBEAT_ACK_TIMEOUT_MS (15m) without ack:
- Send HeartbeatPanicStartMessage
- Get artifact URLs via get_artifact_urls() (same as spot interrupt)
- Execute checkpoint script (PANIC_CHECKPOINT_BUDGET_MS = 5m)
- Upload artifacts in priority order (recency_desc strategy)
- Send HeartbeatPanicCompleteMessage
- Initiate shutdown (same path as spot interrupt)

## Timing Note
SSH activity detected via heartbeat has up to SSH_ACTIVITY_SLACK_MS (60s) slack window before hold-expiry processes it.
This is acceptable - SSH extension is best-effort, not guaranteed.

## Timeline
t=0        Last successful heartbeat ack
t=2m       DEGRADED_CONTROL status (early warning)
t=15m      Agent commits to self-termination, starts checkpoint
t=16-20m   Agent checkpoint complete, self-terminates
t=25m      Control plane forces termination (if agent failed)
