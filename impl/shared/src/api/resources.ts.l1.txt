# resources.types.ts - Layer 1 API Types

## Request/Response Envelopes

### ApiResponse<T>
- data: T
- meta?: object

### ListResponse<T>
- data: T[]
- pagination: PaginationMeta

### ApiError
- code: string
- message: string
- category: ErrorCategory  # See types.ts for full enum
- details?: object
- retryAfterMs?: number    # For rate limit errors

ErrorCategory = 'validation' | 'auth' | 'not_found' | 'conflict' | 'rate_limit' | 'provider' | 'timeout' | 'internal'

Category → HTTP Status mapping:
- validation → 400 Bad Request
- auth → 401 Unauthorized or 403 Forbidden (check details.code)
- not_found → 404 Not Found
- conflict → 409 Conflict
- rate_limit → 429 Too Many Requests
- provider → 502 Bad Gateway (cloud provider error)
- timeout → 504 Gateway Timeout
- internal → 500 Internal Server Error

## Pagination

### Pagination Constants
- DEFAULT_PAGE_LIMIT = 50
- MAX_PAGE_LIMIT = 1000
- CURSOR_TTL_MS = 3600000

### PaginationParams
- limit: number (default: DEFAULT_PAGE_LIMIT, max: MAX_PAGE_LIMIT)
- cursor?: string (opaque base64-encoded JSON: { id: number, created_at: number })

### PaginationMeta
- hasMore: boolean
- cursor: string | null
- limit: number

### Cursor Validation
- Format: base64-encoded JSON { id: number, created_at: number }
- Expiry: 1 hour (CURSOR_TTL_MS)
- Invalid/expired cursor → 400 with error code INVALID_CURSOR

### Sort Order
- Default: created_at DESC
- Empty results: { data: [], pagination: { hasMore: false, cursor: null } }

### FilterParams
- status?: string
- provider?: string
- created_after?: number
- created_before?: number

## Error Codes

- INVALID_INPUT
- INVALID_CURSOR
- UNAUTHORIZED
- NOT_FOUND
- CONFLICT
- RATE_LIMITED
- INTERNAL_ERROR
