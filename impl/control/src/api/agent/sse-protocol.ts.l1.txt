# sse-protocol.ts - SSE Command Protocol

# Note: Timing constants import from ../../../../shared/src/config/timing.ts.l2.txt
# - SSE_RECONNECT_BASE_MS (2s)
# - SSE_RECONNECT_MAX_MS (64s)
# - SSE_MAX_RETRIES (10)
# - HEARTBEAT_ACK_TIMEOUT_MS (30s)
# - PANIC_DIAGNOSTICS_TIMEOUT_MS (2s)

## Stream: Control Plane → Agent
- Endpoint: GET /v1/agent/commands?instance_id=X&allocation_id=Y
- Header: Accept: text/event-stream
- Long-lived unidirectional stream

## Event Types (5)
- start_run: Execute run with file sync
- cancel_run: Terminate running run
- capture_artifacts: Upload artifacts (mid-run or emergency)
- trigger_tailscale: JIT Tailscale installation
- heartbeat_ack: Acknowledge agent heartbeat

## Message Schemas
- StartRunMessage: {type, run_id, allocation_id, command, workdir, files[], artifacts[], run_init?, init_script?, create_snapshot?}
- CancelRunMessage: {type, run_id}
- CaptureArtifactsMessage: {type, run_id}
- TriggerTailscaleMessage: {type}
- HeartbeatAckMessage: {type, control_plane_id}

## Agent Responses (HTTP POST, not SSE)
- POST /v1/agent/logs: {run_id, allocation_id, stream, data, phase?, sync_success?}
- POST /v1/agent/heartbeat: {instance_id, allocation_id?, run_id?, status, tailscale_ip, active_ssh_sessions, tailscale_status}
- stream values: 'stdout' | 'stderr' | 'sync_complete'
- sync_complete triggers CLAIMED→ACTIVE transition

## ConnectionManagement
- Auto-reconnect: exponential backoff SSE_RECONNECT_BASE_MS (2s) → SSE_RECONNECT_MAX_MS (64s), max SSE_MAX_RETRIES (10) retries
- Heartbeat timeout: HEARTBEAT_ACK_TIMEOUT_MS (30s) no ack → agent shutdown
- Panic diagnostics: POST /v1/instances/:id/panic (best-effort, PANIC_DIAGNOSTICS_TIMEOUT_MS / 2s timeout)

## Implementation Notes
- Byte-by-byte reading (Python 8KB buffer deadlocks on ~150 byte messages)
- First heartbeat sent immediately on startup
