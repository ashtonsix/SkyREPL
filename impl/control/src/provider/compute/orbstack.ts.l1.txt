# orbstack.ts - OrbStack Provider (Ch8)

Local testing provider with chaos injection for contract validation.
See L2 for bootstrap generation, VM lifecycle commands, chaos fault injection.

## Capabilities
snapshots: true (instant copy-on-write), spot: false, gpu: false, multiRegion: false, idempotentSpawn: true (by VM name)

## Key Operations
- spawn: Create VM via `orbstack create`, execute bootstrap, idempotent by name
- terminate: Delete VM via `orbstack delete --force`, instant (~1 second)
- createSnapshot: `orbstack snapshot create`, instant (copy-on-write)
- onHeartbeat: Fast reconciliation (30s vs 5 min for cloud), detect chaos drift

## Chaos Testing Framework
- Fault types: capacity_error, spot_interruption, network_partition, drift, api_timeout, auth_failure
- Validates control plane handles failures correctly (retry logic, compensation, state transitions)
- Contract-based testing: OrbStack implements same intent contracts as cloud providers

## Testing Philosophy
Contract as unit of trust. OrbStack implements contracts identically to cloud providers, plus chaos injection. Control plane tested once via OrbStack, not repeated per provider.

## Exports
OrbStackProvider, injectFault, clearFaults
