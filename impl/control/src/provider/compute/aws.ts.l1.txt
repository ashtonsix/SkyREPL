# aws.ts - AWS Provider (Ch8)

Reference implementation with deepest integration. Supports all capabilities including provider-specific workflows.
See L2 for spawn idempotency protocol, bootstrap generation, materialization/projection, lifecycle hooks, provider workflows.

## Capabilities
snapshots: true (AMI), spot: true, gpu: true, multiRegion: true, persistentVolumes: true, warmVolumes: true, hibernation: true, costExplorer: true, idempotentSpawn: true (ClientToken, 24-hour dedup)

## Two-Phase Spawn Protocol
- Phase 1: Reserve in DB (workflow_state='spawn:pending', empty provider_id)
- Phase 2: Execute spawn (call EC2 RunInstances with ClientToken, update provider_id)
- Reconciliation: Find stale spawn:pending, query EC2 by tag, update or mark failed

## Materialization & Projection
- MATERIALIZATION: Generic Instance → AWSInstance (enhanced with AZ, VPC, security groups, volumes)
- PROJECTION: AWSInstance → Generic Instance (strip AWS-specific fields)
- Happens at provider-specific workflow boundaries

## Lifecycle Hooks
- onStartup: Verify credentials, ensure security groups/IAM roles, warm spot price cache
- onHeartbeat: Refresh caches, maintain warm pools, reconcile drift
- onSpotInterruption: Submit emergency-checkpoint + resume-interrupted-run workflows
- onBudgetThreshold: Cleanup non-essential instances

## Provider Workflows
- aws:warm-volume-pool: Pre-create EBS volumes for fast attachment
- aws:spot-fleet: Request spot fleet across multiple AZs (PFO pattern)
