# errors.ts - Provider Error Taxonomy (Ch8)

Standardized error classes that all providers map to. Enables consistent retry logic across providers.
See L2 for error class implementations and retry strategy mapping.

## Error Classes
CapacityError, AuthError, RateLimitError, SpotInterrupted, QuotaExceededError, InvalidSpecError, NotFoundError, InvalidStateError, UnsupportedOperationError

## ProviderError Base
code, category, retryable, retryAfterMs, provider, details

## Error Categories
capacity, auth, rate_limit, validation, not_found, conflict, internal

## Retry Strategy Mapping
- CAPACITY_UNAVAILABLE: retry-with-alternative (3 attempts)
- RATE_LIMITED: same_params with exponential backoff (5 attempts)
- SPOT_INTERRUPTED: fallback to on-demand (1 attempt)
- AUTH_ERROR: not retryable
- QUOTA_EXCEEDED: not retryable

## Helpers
shouldRetryWithAlternative, shouldRetry, calculateBackoff, mapProviderError
