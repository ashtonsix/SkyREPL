# types.ts - Provider Interface & Type Definitions

Core provider interface and registry.
See L2 for contract testing, error retry policies, feature provider integration, provider-specific types.

## ProviderName

aws, lambda, runpod, orbstack

## ProviderInterface

Generic interface: Provider<TInstance, TSnapshot>

Core methods (required): spawn, terminate, list, get, generateBootstrap
Optional methods (capability-gated): createSnapshot, getSnapshotStatus, deleteSnapshot, getSnapshotByName, requestSpotInstance, cancelSpotRequest, getSpotPrices

## ProviderCapabilities

Priority: snapshots, spot, gpu, multiRegion, persistentVolumes
Special: warmVolumes, hibernation, costExplorer, tailscaleNative, idempotentSpawn, customNetworking

## CoreMethodTypes

SpawnOptions, ProviderInstance, ProviderInstanceStatus, ListFilter, BootstrapConfig, BootstrapScript
SnapshotRequest, SnapshotStatus, ProviderSnapshot
SpotOptions, SpotRequest, SpotPriceInfo

## Call Semantics

All methods async but conceptually blocking
Long-running operations return after initial API call
Caller responsible for polling completion
Providers do NOT handle retries (workflow engine handles retry logic)

## FeatureProvider Interface

Feature providers attach to existing instances rather than creating compute
attach(instanceId, config), detach(instanceId), status(instanceId), reconcile?(context), onInstanceHeartbeat?(instanceId)

## Provider-Specific Types

AWSInstance (extends ProviderInstance with EC2 metadata)
RunPodInstance, LambdaInstance, OrbStackInstance
