# contracts.ts - Provider Workflow Contracts (Ch8)

Compile-time specifications defining what workflows must achieve. Not runtime objects.
See L2 for contract validation, registry, constraint enforcement.

## ProviderWorkflowConstraints
maxDurationMs, requiredOutputFields, inputSchema, emittableResources, canSpawnSubworkflows, maxSubworkflowDepth

## AbstractContracts

### SpawnInstanceContract
- Input: spec, region, spot, bootstrap
- Output: instanceId, providerId, ip, status
- Constraints: maxDurationMs=SPAWN_TIMEOUT_MS, emittableResources=['instance']

### TerminateInstanceContract
- Input: providerId, gracePeriodMs
- Output: terminated, finalStatus
- Constraints: maxDurationMs=TERMINATE_TIMEOUT_MS, emittableResources=[]

### CreateSnapshotContract
- Input: providerId, name, spec, initChecksum
- Output: snapshotId, providerSnapshotId, sizeBytes
- Constraints: maxDurationMs=SNAPSHOT_TIMEOUT_MS, emittableResources=['object']

### CleanupManifestContract
- Input: manifestId
- Output: resourcesCleaned, failures
- Constraints: maxDurationMs=600_000, canSpawnSubworkflows=true
- Cleanup order (by priority): Allocations (90) → Runs (80) → Artifacts (70) → Objects (60) → Instances (50) → Snapshots (10)
