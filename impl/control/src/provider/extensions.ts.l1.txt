# extensions.ts - Extension Framework

Hooks, caching, and event bridges for provider-specific behavior.
See L2 for in-memory cache, lifecycle hooks, heartbeat expectations model, TTL configuration, cache invalidation integration.

## ProviderLifecycleHooks

onStartup(): Initialize provider, verify credentials, warm caches
onShutdown(): Flush metrics, cleanup state
onHeartbeat(expectations): Unified heartbeat with tasks and receipts

## HeartbeatExpectationsModel

HeartbeatExpectations: tasks (HeartbeatTask[]), deadline (timestamp)
HeartbeatTask: type (reconcile | health_check | cache_refresh | pool_maintenance), priority, lastRun?, params?
HeartbeatReceipts: receipts (TaskReceipt[]), providerTasks?, workflowsLaunched?, nextHeartbeat?
TaskReceipt: type, status (completed | skipped | deferred | failed), result?, reason?
WorkflowLaunchReport: workflowId, workflowType, trigger, context?

## HookRegistry

registerProviderHooks(provider, hooks), invokeHook(provider, hook, ...args), invokeAllHooks(hook, ...args)
Hooks should not crash control plane (errors caught and logged)

## InMemoryCache

getWithCache<T>(key, ttlMs, fetcher, options?) - Return cached or fetch
CacheOptions: bypassCache? (skip cache read, always fetch fresh)
invalidateCache(pattern), invalidateInstanceCache(instanceId), clearProviderCache(provider)

## TTLConfiguration

Base defaults from timing.ts:
CACHE_SPOT_PRICES_TTL_MS (5m), CACHE_CAPACITY_TTL_MS (1m), CACHE_INSTANCE_STATE_TTL_MS (30s), CACHE_INSTANCE_TYPES_TTL_MS (1h)
Providers can override per-key via byPrefix patterns

## Cache Invalidation Integration

State transitions invalidate cache: spawn complete, terminate complete, allocation state change
Critical operations use bypassCache: orphan detection, allocation claim, budget threshold checks

## Hooks vs Workflows

Hooks: Event-driven, synchronous, lightweight (<1s), trigger workflows
Workflows: Task-driven, asynchronous, long-running, use pattern library
