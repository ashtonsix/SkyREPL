# start-run.ts - Start Run Node

## Exports

```typescript
export interface StartRunInput {
  runId: string;
  instanceId: string;
  allocationId: string;
  command: string;
  env?: Record<string, string>;
}

export interface StartRunOutput {
  started: true;
  syncedAt: number;
}

export const startRunExecutor: NodeExecutor<StartRunInput, StartRunOutput>;
```

## Purpose

Send run command to agent, wait for sync confirmation, transition allocation CLAIMED → ACTIVE.

## Execute Steps

```
async function execute(ctx, input): Promise<StartRunOutput>
  1. Verify instance status:
     - instance = await db.getInstance(input.instanceId)
     - If instance.status !== 'running': throw InvalidStateError

  2. Send start command via SSE:
     - await sse.sendMessage(input.instanceId, {
         type: 'start_run',
         runId: input.runId,
         command: input.command,
         env: input.env,
         workdir: `/work/${input.allocationId}`
       })

  3. Wait for sync_complete event:
     - event = await sse.waitForEvent(input.instanceId, 'sync_complete', {
         runId: input.runId,
         timeout: 600000  // 10 minutes
       })
     - If timeout: throw OperationTimeoutError('Sync timeout')
     - If event.error: throw SyncFailedError(event.error)

  4. Transition allocation CLAIMED → ACTIVE:
     - NOTE: This transition is performed by the control plane, not the agent.
     - The control plane receives the sync_complete log event and updates:
       await db.updateAllocationStatus(input.allocationId, 'ACTIVE')
     - This ensures the state machine is controlled server-side.

  5. Update run record:
     - await db.updateRun(input.runId, {
         status: 'running',
         started_at: Date.now()
       })

  6. Return:
     - { started: true, syncedAt: event.timestamp }
```

## Allocation State Transition: CLAIMED → ACTIVE

**Owner:** Control plane (this node)

**Trigger:** Agent reports sync_complete event

**Implementation:**
- Control plane receives sync_complete via SSE log stream
- This node (start-run) performs the state transition
- Agent does NOT control allocation state - it only reports events

**Why control plane owns this:**
- State machine consistency (single source of truth)
- Agent crash doesn't leave allocation in inconsistent state
- Server can validate transition is allowed

## Compensation

```
async function compensate(ctx, input): Promise<void>
  1. Send cancel command:
     - await sse.sendMessage(input.instanceId, {
         type: 'cancel_run',
         runId: input.runId
       })

  2. Agent will SIGTERM process if running

  3. Update run status:
     - await db.updateRun(input.runId, { status: 'cancelled' })

  4. NOTE: Allocation state NOT reverted (ACTIVE remains ACTIVE)
     - Allocation cleanup handled by transition-hold or failure path
```

## Error Handling

| Error | Source | Response |
|-------|--------|----------|
| Instance not running | Internal | Fail immediately (INVALID_STATE_TRANSITION) |
| SSE send failure | External | Retry with backoff (PROVIDER_API_ERROR) |
| Sync timeout (10 min) | External | Compensate + fail (OPERATION_TIMEOUT) |
| Agent sync failure | External | Compensate + fail (sync error details) |

## Key Calls

- db.getInstance(instanceId)
- db.updateAllocationStatus(allocationId, 'ACTIVE')
- db.updateRun(runId, updates)
- sse.sendMessage(instanceId, { type: 'start_run', ... })
- sse.waitForEvent(instanceId, 'sync_complete', { runId, timeout })
