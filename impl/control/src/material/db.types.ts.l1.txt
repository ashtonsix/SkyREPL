# db.types.ts - Database Record Types and Converters

Raw SQL row types and converters to/from materialized resources.
See L2 for field mappings, JSON parsing, relationship population logic.

## Conventions
- snake_case field names (SQL columns)
- number for SQLite booleans (0/1)
- string for JSON (parsed on read)
- All IDs are INTEGER AUTOINCREMENT

## Record Types
InstanceRecord, RunRecord, AllocationRecord, WorkflowRecord, WorkflowNodeRecord, ManifestRecord, ManifestResourceRecord, ObjectRecord, BlobRecord, UsageRecordDb

## Converters
recordToInstance, instanceToRecord, recordToRun, recordToAllocation, recordToWorkflow, recordToSnapshot, recordToArtifact, recordToFeatureInstallation

## TypeGuards
isInstanceRecord, isRunRecord

## Materialization
- Converters handle field name transformation (snake_case ↔ camelCase)
- Converters handle type coercion (SQLite number ↔ boolean)
- Relationship population happens at materialization layer, not converter layer
- Converters return base objects without relationships (e.g., Instance without features/tailscaleMachine)
