# classifier.ts - Orphan Classification (Ch12)

Classify orphans by origin session and severity. Enables preflight warnings and prioritized investigation.
See L2 for session-based classification logic and preflight warning generation.

## Severity Levels
WARNING (current session), INFO (other sessions), CAUTION (unknown origin)

## Exports
classifyOrphans, parseResourceName, generateResourceName, preflightOrphanCheck

## Classification Rules
- Current session: created after sessionStartTime OR age < threshold (default 1h) → WARNING
- Other sessions: age >= threshold, parseable name → INFO
- Unknown: unparseable name (doesn't match repl-* pattern) → CAUTION

## Resource Name Format
repl-{controlId}-{manifestSlug}-{instanceSlug}
Tailscale: repl-{controlId}-{instanceSlug}

## OrphanClassification Structure
currentSession, otherSessions, unknown, stats (total, byProvider, bySeverity)

## Preflight Warnings
Check for orphans before launch-run, terminate-all, cleanup operations. Return warnings for CLI display.

## Use Cases
Preflight warnings, prioritize recent orphans, filter by severity, inform automatic cleanup decisions (INFO = safe to terminate)

## Materialization
Classification is computed on-demand. Orphan data sourced from provider APIs and DB.
