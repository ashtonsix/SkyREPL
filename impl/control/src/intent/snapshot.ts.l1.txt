# snapshot.ts - Create Snapshot Intent (Ch7)

Create provider snapshot for fast instance provisioning. Snapshots capture disk state, NOT workdir state.
See L2 for prepare-instance agent protocol, provider snapshot API, wait-for-snapshot polling.

## Snapshot Scope
- Captures: Root filesystem, installed packages, system configuration
- Does NOT capture: Workdir state (/work/run_*), ephemeral data, run-specific artifacts
- Key principle: Workdir preserved via artifacts, NOT snapshots

## Workflow Graph
validate-snapshot-request → prepare-instance → create-provider-snapshot → wait-for-snapshot → register-snapshot-object

## Key Nodes
- validate-snapshot-request: Check provider supports snapshots, no concurrent snapshot
- prepare-instance: Send prepare_snapshot command to agent (sync, clear temp/workdir)
- create-provider-snapshot: Call provider.createSnapshot() with tags
- wait-for-snapshot: Poll for completion (30 min timeout)
- register-snapshot-object: Create object record with type='snapshot', emit to manifest

## ProviderAbstraction
SnapshotCapability: createSnapshot, getSnapshotStatus, deleteSnapshot

## Exports
SnapshotInput, SnapshotOutput, snapshot, snapshotWorkflow
