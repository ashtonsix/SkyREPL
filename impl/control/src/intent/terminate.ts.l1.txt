# terminate.ts - Terminate Instance Intent (Ch7)

Graceful instance shutdown with allocation draining and SSH session handling.
See L2 for allocation drain logic, SSH timeout handling, feature cleanup, and provider termination.

## Workflow Graph
validate-instance → drain-allocations → drain-ssh-sessions → cleanup-features → terminate-provider → cleanup-records

## Key Nodes
- validate-instance: Check instance exists, acquire termination lock (workflow_state = 'terminate:draining')
- drain-allocations: Cancel active runs, transition allocations to COMPLETE
- drain-ssh-sessions: Notify SSH users, wait for graceful disconnect, force remaining
- cleanup-features: Parallel teardown of FeatureInstallations (PFO pattern)
- terminate-provider: Call provider.terminate(), handle NOT_FOUND gracefully
- cleanup-records: Update workflow_state = 'terminate:complete', close usage records
