# 001_initial.sql - Initial Database Schema

SkyREPL control plane database schema. All DDL copied from spec chapters.
See L2 for full table definitions, constraints, indexes, foreign keys.

## Tables
- instances: provider, provider_id, spec, region, ip, workflow_state, spawn_idempotency_key, is_spot, init_checksum
- runs: command, workdir, max_duration_ms, workflow_state, exit_code, create_snapshot, spot_interrupted
- allocations: run_id, instance_id, status, workdir, debug_hold_until, created_at, updated_at
- workflows: type, parent_workflow_id, status, current_node, input_json, output_json, manifest_id, idempotency_key
- workflow_nodes: workflow_id, node_id, node_type, status, input_json, output_json, attempt
- manifests: workflow_id, status, default_cleanup_priority, retention_ms, expires_at
- manifest_resources: manifest_id, resource_type, resource_id, cleanup_priority
- objects: type, blob_id, provider, provider_object_id, metadata_json, current_manifest_id
- object_tags: object_id, tag (format: "key:value")
- blobs: bucket, checksum, checksum_bytes, s3_key, payload, size_bytes, last_referenced_at
- usage_records: instance_id, allocation_id, run_id, provider, spec, started_at, finished_at, duration_ms
- instance_panic_logs: instance_id, reason, last_state_json, diagnostics_json, timestamp
- idempotency_keys: key, endpoint, params_hash, response_body, workflow_id
- orphan_whitelist: provider, provider_id, resource_type, reason, acknowledged_by
- api_keys: key_hash, name, permissions, created_at, last_used_at
- rate_limit_log: key, endpoint, timestamp, window_ms
- orphan_scans: provider, scanned_at, orphans_found, orphan_ids
- manifest_cleanup_state: manifest_id, status, current_step, progress_json

## Key Indexes
- idx_instances_spawn_key: UNIQUE on spawn_idempotency_key (prevents duplicate spawns)
- idx_allocations_available: Warm pool query optimization (status='AVAILABLE')
- idx_blobs_dedup: CAS dedup lookup (bucket, checksum)
- idx_workflows_idempotency: UNIQUE on idempotency_key
