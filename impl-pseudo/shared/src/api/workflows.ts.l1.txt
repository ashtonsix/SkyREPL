# workflows.ts - Workflow API Types

Workflow engine types: blueprints, runtime state, node execution context, pattern configs.
See L2 for blueprint schema, node executor interface, pattern configuration, context methods.

## Workflow Types
WorkflowType: launch-run | terminate-instance | create-snapshot | cleanup-manifest | extensible

## Blueprint Components
WorkflowBlueprint, NodeDefinition, RetryPolicy, AllowedPattern, PatternConstraints, JsonSchema

## Runtime Types
WorkflowSubmission, WorkflowSubmissionResult, WorkflowProgress, WorkflowEvent, NodeExecutionState

## Node Context
NodeContext provides: identity, input/output, resource management, pattern application, subworkflow spawning, node communication, logging, utilities

## Pattern Configs
InsertAndReconvergeConfig, ConditionalBranchConfig, ParallelFanOutConfig, RetryWithAlternativeConfig, InlineNodeDef

## Executor Interface
NodeExecutor with execute, compensate, validate methods

## Intent Types
LaunchRunInput/Output, TerminateInstanceInput/Output, CreateSnapshotInput/Output, CleanupManifestInput/Output

## Materialization
Shared types used by control plane (engine) and CLI (submission).
