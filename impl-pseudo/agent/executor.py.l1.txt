# executor.py - Run Executor (Ch11)

Handles start_run, cancel_run, prepare_snapshot commands. Manages subprocess lifecycle, file sync, and command acknowledgment.
See L2 for command deduplication, workdir cleanup, and sync_complete protocol.

## Exports
RunExecutor (start_run, cancel_run, prepare_snapshot, capture_artifacts)

## start_run Sequence
1. Dedup check + ack
2. Clean workdir (critical for warm pool security)
3. Download files (verify checksums, request URL refresh if transfer >10min)
4. Execute init script (if provided, timeout 5min)
5. Send sync_complete (bypasses batching via flush_and_send)
6. Execute command (subprocess) or skip if snapshot-only mode
7. Stream stdout/stderr (buffered, flush every 100ms)
8. Wait for completion
9. Send run_complete
10. Collect and upload artifacts (timeout 30min)

## cancel_run Sequence
Send SIGTERM, wait 10s, SIGKILL if still running

## prepare_snapshot Sequence
Sync filesystems, clear temp files, clear workdir (no run state in snapshots), flush buffers, send prepare_snapshot_ack

## Command Acknowledgment
- Track last 100 processed_command_ids (deque, maxlen=100)
- Queue pending_command_acks, send via heartbeat
- Clear pending_command_acks after successful heartbeat

## State Tracking
current_run_id, current_process, current_allocation_id, interrupted, pending_command_acks, processed_command_ids

## Materialization
Subprocess management (no DB storage). Sends messages to control plane via HTTP POST.
