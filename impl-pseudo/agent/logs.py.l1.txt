# logs.py - Log Collection and Batching

Two-level batching: local buffer (100ms) → network batch (5s or 64KB).
See L2 for batching strategy, sync_complete bypass, buffer overflow handling, sequence IDs.

## Batching Levels
- Level 1: Local buffer flush every 100ms
- Level 2: Network send every 5s OR 64KB threshold
- Special: sync_complete bypasses both (sent immediately)

## Exports
LogBuffer, log_buffer, log_flusher_thread, send_log_batch, flush_and_send, flush_all, flush_all_logs, get_dropped_logs_count, get_log_buffer_size

## State Tracking
- batch_sequence_id: Monotonic counter for reordering
- dropped_logs_count: Total dropped (included in heartbeat)
- total_queued_bytes: Buffer size tracking

## ErrorHandling
- Network failure: Retry with exponential backoff (1s → 32s max)
- Buffer overflow (>1MB): Drop oldest, insert warning, increment counter

## Materialization
POST /v1/agent/logs with LogMessage[] payload.
