# instance.ts - Instance Materialization (Ch3)

## Material Backing
- Primary: instances table
- Secondary: objects table (feature_installation, tailscale_machine types)
- External: Provider API (IP, status, metadata) - cached 30s TTL

## Key Fields
- provider: aws | lambda | runpod | orbstack
- provider_id: External ID (empty during spawn:pending)
- spec: Instance type (e.g., "g5.xlarge")
- region: Provider region (default 'local' for OrbStack)
- ip: Cached from external state
- is_spot: Spot vs on-demand
- init_checksum: For warm pool matching
- last_heartbeat: Agent liveness tracking

## Relationships
- Has many: Allocations, FeatureInstallations
- Has one: TailscaleMachine (optional)
- Belongs to: Manifest (optional)

## Materialization Steps
1. Get base record from instances table
2. Get feature installations: objects WHERE type='feature_installation' AND tags.instance_id
3. Get Tailscale machine: objects WHERE type='tailscale_machine' AND tags.instance_id
4. Get external state (if stale): provider.getInstance(provider_id)
5. Combine, preferring fresh external data (IP, status)

## Lifecycle States
- spawn:pending → launch-run:provisioning → launch-run:bootstrapping → launch-run:complete
- terminate:draining → terminate:complete
- Any state → :error suffix on failure
