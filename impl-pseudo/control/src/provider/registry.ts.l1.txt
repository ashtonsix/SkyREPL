# registry.ts - Provider Registry & Discovery

Single registry for provider discovery, used by both runtime and tests.
See L2 for lazy-loading, caching, parametric testing, credential isolation.

## Provider Registry

Static registry with lazy-load functions:
aws, lambda, runpod, orbstack

## RuntimeAccess

getProvider(name), getAllProviders(), isProviderRegistered(name), clearProviderCache()

## Dynamic Registration

registerProvider(name, registration), unregisterProvider(name)
Enables plugin-based provider extensions (future)

## FeatureProviderRegistry

Separate registry for features (tailscale, wandb, vscode-server)
getFeatureProvider(name), getAllFeatureProviders(), isFeatureProviderRegistered(name), clearFeatureProviderCache()

## TestUtilities

getTestableProviders() - OrbStack always included, cloud providers gated by credentials
getTestableFeatureProviders() - Similar logic for features
TEST_PROVIDERS env var for explicit selection

## Credential Isolation

Each provider loads credentials independently, no cross-provider sharing
Lazy configuration loading reduces startup time
Validation on first access, cached for session
Separate cache namespaces prevent cross-provider leakage
