# compensation.ts - Compensation Framework (Ch6)

Single-node compensation for failed workflow nodes. Two-phase spawn protocol prevents orphans.
See L2 for retry logic, timeout handling, two-phase spawn implementation, reconciliation.

## Core Principle
Compensate ONE node only (the failed node). NO cascade through completed nodes. Manifest policies handle broader cleanup.

## Compensation vs Teardown
- COMPENSATION: error correction DURING workflow (single node)
- TEARDOWN: scheduled cleanup AFTER workflow (manifest expiry)

## Two-Phase Spawn
- Phase 1: Create DB record + manifest entry BEFORE cloud API call
- Phase 2: Execute cloud API call, update DB with provider_id
- Property: Kill at any step → no orphans
- Reconciliation finds instances if DB update fails after cloud call

## Compensation Timeout and Retry
- COMPENSATION_TIMEOUT_MS: 5 minutes per handler (from timing.ts)
- Retry up to 3 times for retryable errors (RATE_LIMITED, NETWORK_ERROR, PROVIDER_API_ERROR)
- Non-retryable errors fail immediately (VALIDATION_ERROR, NOT_FOUND)
- Backoff schedule: 1s, 2s, 4s
