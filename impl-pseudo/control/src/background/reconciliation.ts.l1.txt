# reconciliation.ts - Reconciliation Task

Background task (every 60s): syncs instance state, detects degraded/stale heartbeats, recovers stuck allocations.
See L2 for instance state sync, degraded detection (2m), heartbeat timeout (25m with graceful panic), stuck CLAIMED recovery, orphan AVAILABLE cleanup.

## Cycle Interval
RECONCILIATION_INTERVAL_MS (60s)

## Tasks
- syncInstanceState: Query provider APIs, update workflow_state on drift
- detectDegradedHeartbeats: Mark degraded at 2m (early warning before 25m force-term)
- checkHeartbeatTimeouts: Force-terminate at 25m, check graceful panic checkpoint first
- checkPendingSpawns: Poll spawn:pending instances for completion
- recoverStuckClaimedAllocations: Fail CLAIMED >10m with explicit status guard
- cleanupOrphanAvailableAllocations: Fail AVAILABLE with dead instances, prevent warm pool pollution

## Graceful Panic Detection
Before marking ACTIVE→FAILED, check for checkpoint object created after last_heartbeat.
If found: COMPLETE (checkpoint preserved), else: FAILED (data lost).

## Status Guards
Stuck allocation recovery uses explicit status guards (UPDATE WHERE status='CLAIMED') to prevent race conditions.

## Exports
runReconciliationCycle, startReconciliationLoop, syncInstanceState, detectDegradedHeartbeats, checkHeartbeatTimeouts, checkPendingSpawns, recoverStuckClaimedAllocations, cleanupOrphanAvailableAllocations, checkGracefulPanic

## Materialization
Background task, no persistent state beyond DB updates.
