# garbage-collection.ts - Garbage Collection Task (Ch15)

Background task to clean up orphaned blobs, merge duplicates, and expire objects/keys.
See L2 for incremental execution, CAS-guarded dedup merger, and S3 deletion logic.

## Purpose
DB cascade can't reach S3, so GC is needed for S3 cleanup. Runs every 24h.

## Primary Task: Blob GC
- Find orphaned blobs: no referencing objects, last_referenced_at < now - 24h
- Delete from S3 (if s3_key exists), then DELETE FROM blobs
- Batch size: 500 blobs/cycle
- Update last_referenced_at for recently accessed blobs (prevents premature deletion)

## Secondary Task: Dedup Merger
- Find duplicate blobs (same bucket + checksum) for dedupable buckets (run-files, artifacts)
- Merge: keep oldest, UPDATE objects to point to kept blob, DELETE duplicates
- CAS guard: re-verify duplicate set in transaction to prevent concurrent merge corruption

## Tertiary Task: Object Expiry
- Find expired objects (expires_at < now), DELETE FROM objects
- Blobs cleaned by primary GC task (grace period applies)

## Quaternary Task: Idempotency Key Expiry
- Find expired idempotency keys (expires_at < now), DELETE FROM idempotency_keys
- Prevents unbounded table growth

## Metrics
blobs_deleted, s3_objects_deleted, bytes_freed, duplicates_merged, objects_expired, idempotency_keys_deleted

## Materialization
Background loop (setInterval). Updates blobs, objects, idempotency_keys tables and deletes from S3.
