# external.ts - Provider State Caching

In-memory TTL cache for ephemeral provider state (not persisted to DB).
See L2 for cache implementation, invalidation hooks, bypass semantics.

## Cache Keys
Format: {provider}:{type}:{region}:{spec} or instance:{id}:external

## TTL Defaults
- Spot prices: 5 minutes
- Instance state: 30 seconds
- Snapshot state: 5 minutes

## Exports
getWithCache, invalidateInstanceCache, invalidateProviderCache, invalidateCacheByKey

## Cache Bypass
Critical operations use bypassCache=true option to skip cache read (still updates cache after fetch).
Used by: orphan detection, allocation claims, budget checks.

## Invalidation Hooks
- invalidateInstanceCache: Called on spawn/terminate/allocation state change
- invalidateProviderCache: Pattern-based invalidation (e.g., 'instance:*')
- invalidateCacheByKey: Direct key targeting

## Staleness Risks
Up to 30s staleness window for instance state. Mitigations: bypass flag, invalidation hooks, grace periods.

## Materialization
In-memory Map, cleared on restart.
