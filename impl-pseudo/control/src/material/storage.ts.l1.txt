# storage.ts - Object Storage (Ch2)

Blob storage with inline (≤64KB) vs S3 (>64KB) routing. Deduplication is opt-in per bucket.
See L2 for presigned URL generation, multipart uploads, deduplication race handling, blob GC.

## Thresholds
- INLINE_THRESHOLD: 64KB (stored in DB payload column)
- LARGE_FILE_THRESHOLD: 100MB (triggers multipart)
- PRESIGNED_URL_TTL: 4h (from timing.ts)

## Exports
generateUploadUrl, generateDownloadUrl, storeInline, getInline, uploadToS3, downloadFromS3, deleteFromS3, generateMultipartUploadUrls, completeMultipartUpload, abortMultipartUpload, checkBlobsExist, createBlobWithDedup, prepareFilesForRun, appendLogData, getLogUpdates, runBlobGC, cleanupManifestObjects

## Deduplication
- Opt-in per bucket: run-files, artifacts, snapshots (dedupable); logs (non-dedupable)
- No UNIQUE constraint (allows race duplicates, GC merges later)
- checkBlobsExist returns missing checksums + existing blob IDs

## Multipart Uploads
- Required for files >5GB, recommended for >100MB
- Part size: min 5MB (except last), max 5GB
- Max parts: 10,000

## Materialization
DB blobs table + S3 bucket for large objects.
