# db.ts - Database Layer

SQLite database layer with WAL mode, migrations, and query helpers.
See L2 for connection management, query helpers, transaction retry with jittered backoff, atomic claim pattern, blob GC operations.

## Connection

initDatabase(path), getDatabase(), closeDatabase()
WAL mode, foreign keys enabled, busy_timeout 5000ms

## Migrations

runMigrations(), getMigrationVersion()
Apply schema migrations from version files

## QueryHelpers

queryOne<T>(sql, params), queryMany<T>(sql, params), execute(sql, params), transaction<T>(fn, retries?)
Transaction retry with SQLITE_BUSY handling using calculateBackoff() for jittered exponential backoff

## InstanceOperations

getInstance(id), getInstanceByProviderId(provider, providerId), createInstance(data), updateInstance(id, updates), listInstances(filter?)

## RunOperations

getRun(id), createRun(data), updateRun(id, updates), listRuns(filter?)

## AllocationOperations

getAllocation(id), createAllocation(data), updateAllocationStatus(id, status), claimAllocation(id, runId), findAvailableAllocation(spec)

claimAllocation concurrency: Two-phase atomic transition pattern (SELECT with updated_at, UPDATE with WHERE updated_at = original)
Returns null if claim fails (not found, wrong state, or race lost)

## WorkflowOperations

getWorkflow(id), createWorkflow(data), updateWorkflow(id, updates), getWorkflowNodes(workflowId), createWorkflowNode(data), updateWorkflowNode(id, updates)

## ManifestOperations

getManifest(id), createManifest(workflowId, options?), sealManifest(id), addResourceToManifest(manifestId, resourceType, resourceId, options?), getManifestResources(manifestId), deleteManifest(id), listExpiredManifests(cutoffTime)

addResourceToManifest invariant: Throws if manifest.status !== 'DRAFT' (except A13 recovery with allowRecovery=true)
ownerType/ownerId track provenance: 'workflow' | 'user' | 'system'

## B21 Cleanup Processed Tracking

manifest_resources.cleanup_processed_at prevents double-processing of blob reference decrements

## BlobOperations

getBlob(id), createBlob(data), findBlobByChecksum(bucket, checksum), updateBlobLastReferenced(id)
CAS dedup is OPTIONAL (opt-in per bucket): 'run-files', 'artifacts' dedupable; 'logs' always create new

## ObjectOperations

getObject(id), createObject(data), addObjectTag(objectId, key, value), findObjectByTag(key, value)
All objects require blob_id (NOT NULL)

## Warm Pool

findWarmAllocation(spec, initChecksum?), getWarmPoolStats()

## OrphanOperations

getTrackedInstanceIds(), getActiveManifestIds(), recordOrphanScan(result), addToWhitelist(providerId, reason), isWhitelisted(providerId)

## DeleteOperations

deleteInstance(id), deleteRun(id), deleteAllocation(id), deleteObject(id), deleteBlob(id), deleteWorkflow(id)
Validation: instance (no active allocations), run (completed/failed), allocation (COMPLETE/FAILED), blob (no referencing objects)

## BackgroundBlobGC

findOrphanedBlobs(cutoff24hAgo) - No referencing objects + 24h grace period
deleteBlobBatch(blobIds) - Delete after S3 cleanup
GC required because DB CASCADE cannot reach S3
