# ssh_fallback.sh - SSH Config Catch-All Fallback

## Purpose
Handles SSH connections to allocations not in current SSH config:
- Stale config (allocation terminated/recreated)
- New allocations before next sync
- Manual connections with auto-discovery

## Input
- %n from SSH config: hostname like "repl-abc123"

## Slug Extraction
- Strip "repl-" prefix from hostname
- Extract slug: "repl-abc123" → "abc123"
- Validate format (alphanumeric, reasonable length)

## Control Plane Query
GET /v1/allocations?slug_prefix=${SLUG}
- Uses CLI auth context (token from ~/.repl/config)
- Filter: status IN ('CLAIMED', 'ACTIVE')
- Returns matching allocations with instance details

## Response Handling

### Case 1: Zero Matches
- Error message: "No active allocation found for 'repl-abc123'"
- Suggest recent allocations: "Did you mean?"
  - Query GET /v1/allocations?limit=5&order=updated_at DESC
  - Display: slug, status, last_updated relative time
- Exit code: 1

### Case 2: One Match
- Warning to stderr: "SSH config is stale. Run 'repl ssh-sync' to update."
- Extract allocation details (instance_id, user, workdir)
- Query GET /v1/instances/:id for IP address
- Delegate to ssh_proxy.sh with proper parameters:
  - Instance IP
  - User
  - Workdir (for RemoteCommand)
- Exit code: 0 (success) or ssh_proxy.sh exit code

### Case 3: Multiple Matches
- Error message: "Ambiguous allocation slug 'abc123'. Multiple matches:"
- List all matches:
  - Full slug
  - Status
  - Run ID
  - Instance ID
  - Last updated time
- Suggest: "Use full allocation slug to disambiguate"
- Exit code: 1

## ErrorHandling
- Network errors: "Failed to reach control plane at ${API_URL}"
- Auth errors: "Authentication failed. Run 'repl auth login'"
- API errors: Display error message from response
- Timeout: 10 second connection timeout
- All errors to stderr, exit code: 1

## Exit Codes
- 0: Success (delegated to ssh_proxy.sh)
- 1: User error (not found, ambiguous, stale config)
- 2: System error (network, auth, API)

## Integration
- Called by SSH via ProxyCommand: ProxyCommand ssh_fallback.sh %n
- Installed to ~/.repl/bin/ during CLI setup
- Must be executable
- Uses same auth/config as CLI
