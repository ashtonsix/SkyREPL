# preflight.ts - Pre-flight Checks

## Integration
- GET /v1/preflight before workflow submission
- Called automatically by `repl run`, `repl instance terminate`, etc.

## Warning Types
- orphans: Untracked cloud resources (cost estimate)
- budget: Approaching or exceeded budget limits
- credentials: Provider credentials expiring soon
- stale_allocations: COMPLETE allocations with debug_hold_until older than 24h
- provider_issues: Provider API errors or quota warnings
- low_disk_space: Warm pool instances with <20% free disk
- shared_instance: Target instance has other active allocations
- degraded_instance: Instance has degraded control plane connection (>2m since last heartbeat ack)

## Warning Levels
- info: Display only (orphans from other sessions, shared_instance notice)
- warning: Prompt for confirmation (budget near limit, low_disk_space, degraded_instance)
- critical: Block unless --force (credentials expired)

## Display Format
- Group warnings by type
- Show cost impact where applicable
- Include recommended actions
- For low_disk_space: Show instance ID, free %, suggest cleanup
- For shared_instance: List other allocations, their status and duration
- For degraded_instance: Show instance ID, time since last ack, warn of potential data loss

## Bypass
- --yes: Skip confirmation prompts, proceed on warnings
- --no-preflight: Skip preflight entirely (for automation)
- Neither bypasses critical errors
