--- DRIFT-BAD ---

DB-1  No base-36 slug conversion anywhere (spec Ch02, docs/design/rationale-material.txt)
      Spec: INTEGER AUTOINCREMENT PKs; user-facing as base-36 slugs.
      Rationale doc: `id.toString(36)` for display, `parseInt(slug, 36)` to
      parse. Pure 1:1 conversion. Benefits: shorter than UUIDs, sortable,
      human-friendly. Example: 12345 → "9ix".
      Impl: Raw integer IDs used everywhere — wire format, CLI display,
      agent messages. No slug conversion exists in the codebase.
      Files: shared/src/api/agent.ts:31,52,70, agent/artifacts.py:29,
             cli/src/commands/*.ts (all display raw ints)
      Impact: HIGH. Pervasive, systemic (predates G2). Where there's smoke
      there's fire — drift found in one place, so probably exists in more.
      Action: FIX IMPL (THOROUGH) —
        1. Wire format: integer IDs (already correct, keep as-is)
        2. CLI display: add slug conversion (id.toString(36)) everywhere
        3. CLI parsing: accept slugs in commands (parseInt(slug, 36))
        4. API responses: consider adding a `slug` field alongside `id`
        5. Add integrity check in scripts/ to flag primary key violations
           with pre-commit hook
        6. Sweep all wire formats to verify integer PKs throughout

DB-2  Heartbeat thresholds not env-configurable (spec 11.7:446)
      Spec: SKYREPL_HEARTBEAT_DEGRADED_MS env var configurable.
      Impl: heartbeat.py:28 hard-codes 2*60*1000.
      Action: FIX IMPL — read from env vars.

DB-3  tailscale_ip not nullable in shared schema (spec 11.4:226)
      Spec: string | null. Impl: Type.String() (not nullable).
      File: shared/src/api/agent.ts:139
      Action: FIX IMPL — use Nullable(Type.String()).

DB-4  Pending acks cleared before confirmed delivery (spec 11.4.5:248)
      Spec: clear acks after control plane confirms.
      Impl: executor.py:136-141 get_and_clear_pending_acks() clears
      immediately. If heartbeat POST fails, ack IDs are lost.
      Impact: Unnecessary command retries (dedup buffer protects execution).
      Action: FIX IMPL — clear acks only after 200 response.

DB-5  Env var prefix: spec uses REPL_, impl uses SKYREPL_ (spec 11.7:446-454)
      Decision: SKYREPL_ preferred. Agent already uses SKYREPL_.
      Control plane uses REPL_ per spec.
      Action: SPEC FIX — update spec to use SKYREPL_ prefix. Align control
      plane env var reading to accept SKYREPL_ prefix.

DB-12 Artifact upload: base64 POST, not presigned URLs (spec 11.11:548)
      Reclassified from DM-1. The control plane must provide a clean
      migration path to a world where the agent doesn't know/care whether
      it's uploading to S3 or control plane — should always be presigned URL.
      Current base64 POST bakes in knowledge of storage backend.
      Action: FIX IMPL (INTERMISSION) — design presigned URL interface.
      Fine to defer the S3 implementation, but the agent-side interface
      must be presigned-URL from the start so it doesn't need to change later.

DB-13 Batch JSON artifact upload (spec 11.4:177)
      Reclassified from DM-2. Batch upload for many small artifacts is an
      interesting optimization, but premature — adds complexity to the
      protocol before the single-file path is proven. Revert to single-file
      upload now; add backlog entry to reconsider batch upload later.
      Action: FIX IMPL (NOW) — revert to single-artifact upload per POST.
      Add #BL-17 to reconsider batch artifact upload.
