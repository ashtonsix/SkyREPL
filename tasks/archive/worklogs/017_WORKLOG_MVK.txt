PURPOSE
-------
After Slices 1-2 + Intermission, the engine works but the user surface is
incomplete. This worklog closes the gap to a self-contained MVK: a human can
cd into demo/, run commands, see output, list resources, cancel runs, and
manage the control plane — without needing curl or manual env vars.

Once MVK lands, development shifts from slice-by-slice to parallel epics
(covering spec breadth and ongoing development beyond core functionality).

DESIGN DECISIONS
----------------
- TOML config: Bun.TOML built-in, no new deps
- Profile = named TOML table; [default] when no -p flag
- Resolution: CLI flag > profile > env file > hardcoded default
- Control plane as background process with PID file in ~/.repl/
- host.internal detection scoped to OrbStack provider only
- Subcommand routing: `repl <noun> <verb>` (run list, run cancel, instance list)
- Cancel goes through workflow (POST /v1/workflows/:id/cancel), not run
- DB lives in CWD (skyrepl-control.db), reset cleans it
- All non-error CLI output to stdout; only errors to stderr
