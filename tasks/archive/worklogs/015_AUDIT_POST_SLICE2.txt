1.1 COMMUNICATION PATTERN SMELLS
────────────────────────────────────────────────────────────────────────────────

SM-01: DB polling in AgentBridge waitForEvent()                        [HIGH]
  File: control/src/api/agent.ts:596-632
  Issue: Polls DB every 1s for sync_complete/run_complete. N concurrent
    runs = 2N queries/sec for pure observation. 1s latency floor.
  Fix: In-process EventEmitter fired from agent HTTP handlers on state
    change. waitForEvent() awaits emitter with timeout.

SM-02: Unbounded pending command queue                                 [HIGH]
  File: control/src/api/sse-protocol.ts:81
  Issue: pendingCommands Map has no size limit. Agent offline + workflows
    submitting commands = unbounded memory growth. Stale commands replayed
    on reconnect (e.g., start_run for timed-out workflow).
  Fix: Cap at 50/instance, FIFO evict with logging. Filter stale commands
    on drain by checking parent workflow status.

SM-03: Engine polls for running nodes instead of awaiting promises     [MEDIUM]
  File: control/src/workflow/engine.ts:258-261
  Issue: executeLoop sleeps 100ms then re-reads all nodes from DB when
    waiting — but it already holds the Promise.all from executing nodes.
    Adds 100ms latency + unnecessary DB reads per inter-node transition.
  Fix: Await the existing Promise.all. Only poll on crash-recovered
    workflows (where promises are not in memory).

SM-04: CLI polls status redundantly with WebSocket                     [MEDIUM]
  File: cli/src/commands/run.ts:76-92
  Issue: Opens WebSocket for logs AND polls GET /status every 2s. WS
    already provides status events. Double load on control plane.
  Fix: Trust WS status event as primary. Poll at 30s as fallback only.

SM-05: Agent SSE byte-by-byte reading                                  [MEDIUM]
  File: agent/sse.py (entire file)
  Issue: response.read(1) workaround for Python 8KB buffer deadlock.
    Extremely slow for high message volume.
  Fix: socket.makefile(buffering=1) for line-buffered reads, or read in
    256-byte chunks and buffer locally. Stdlib-only constraint respected.

SM-06: Duplicate _http_post() across 3 agent modules                   [LOW]
  Files: agent/heartbeat.py:78, agent/executor.py:56, agent/logs.py:63
  Issue: Identical function copy-pasted. Bug fixes/auth changes need 3
    updates. No connection pooling.
  Fix: Extract to shared http_client.py module.

1.3 STATE MANAGEMENT SMELLS
────────────────────────────────────────────────────────────────────────────────

SM-11: Compensation bypasses state machine with raw SQL                [HIGH]
  File: workflow/nodes/claim-allocation.ts:93-98
  Issue: compensate() does raw UPDATE to transition CLAIMED→AVAILABLE,
    bypassing atomic guard in state-transitions.ts. CLAIMED→AVAILABLE is
    NOT a valid transition. No concurrency protection.
  Fix: Add releaseAllocation() to state-transitions.ts with atomic guard,
    or compensate via failAllocation() + create new AVAILABLE.

SM-12: Cold-path creates allocation directly in CLAIMED state          [HIGH]
  File: workflow/nodes/create-allocation.ts:70-79
  Issue: createAllocation({ status: "CLAIMED" }) skips AVAILABLE state.
    Reconciliation can't distinguish stalled warm claims from in-progress
    cold creates.
  Fix: Create AVAILABLE then immediately claim in same transaction, or
    add source:'warm'|'cold' column for reconciliation.

SM-13: Inconsistent HTTP error responses                               [MEDIUM]
  File: api/routes.ts:132-140, 157-167, 192-200
  Issue: Validation errors return 200 OK (plain object). Not-found errors
    correctly return 404 (new Response). 200 for validation errors is
    almost certainly unintentional.
  Fix: Use ElysiaJS set.status consistently, or global error middleware
    with httpStatusForCategory().

SM-14: Manual input validation instead of TypeBox                      [MEDIUM]
  File: api/routes.ts:243-297
  Issue: launch-run manually validates fields. LaunchRunRequestSchema
    TypeBox schema already exists. Manual code misses constraints
    (minLength, exclusiveMinimum).
  Fix: Use ElysiaJS .body(LaunchRunRequestSchema) — native TypeBox
    support with auto 400 on validation failure.

SM-15: Heartbeat ack race between HTTP and SSE paths                   [LOW]
  File: agent/heartbeat.py:146-149, 216-236
  Issue: _last_ack_time written without lock from HTTP path, with lock
    from SSE path. CPython GIL makes it safe in practice.
  Fix: Acquire _lock in both paths for clarity.

1.5 CONCURRENCY / TIMING SMELLS
────────────────────────────────────────────────────────────────────────────────

SM-24: Timer leak in createTimeout()                                   [HIGH]
  File: workflow/engine.ts:1128-1142
  Issue: setTimeout never cleared when node wins Promise.race. Orphaned
    timers accumulate (9 per workflow × timeout duration). Memory leak
    plus unhandled rejection handlers.
  Fix: Return timer handle, clearTimeout() on node completion. Or use
    AbortController pattern.

SM-25: executeLoop fire-and-forget — engine crash = zombie workflow    [HIGH]
  File: workflow/engine.ts:207-213
  Issue: .catch(console.error) on executeLoop. Engine-level error leaves
    workflow in 'running' forever. Only recovered on restart.
  Fix: Catch → failWorkflow() with INTERNAL_ERROR code.

SM-26: SSE workflow stream cancel passes {} as WebSocket               [MEDIUM]
  File: api/sse-protocol.ts:573
  Issue: unsubscribeFromWorkflow(workflowId, {} as WebSocket) is a no-op.
    Subscriber set grows with each SSE connection, never cleaned.
    Self-healing via error on broadcast + catch-delete, but noisy.
  Fix: Track SSE controllers separately. Add unsubscribeSSE method.

SM-27: No cleanup of log replay buffers / sequence counters            [MEDIUM]
  File: api/sse-protocol.ts:93-96
  Issue: logSequenceCounters, logReplayBuffers, logSubscribers maps
    grow monotonically. Each completed run leaves 500-entry replay buffer.
  Fix: Add cleanupRun(runId) called from finalize or workflow completion.

SM-28: Log buffer overflow uses list.pop(0) — O(n)                    [MEDIUM]
  File: agent/logs.py:120-121
  Issue: _send_queue.pop(0) in loop is O(n²) under backpressure.
  Fix: Use collections.deque (popleft is O(1)).

SM-29: SSE replay buffer uses splice — O(n)                           [MEDIUM]
  File: api/sse-protocol.ts:265-267
  Issue: buffer.splice(0, excess) on every log broadcast at buffer cap.
  Fix: Ring buffer with write index and modular arithmetic.

SM-30: warmPoolReconciliation uses dynamic imports                     [LOW]
  File: main.ts:167-169
  Issue: await import() inside function called on interval. Hides
    dependencies. Likely workaround for circular imports.
  Fix: Move imports to top of file. If circular, refactor to break cycle.

SM-31: terminateOrphanedVMs kills ALL VMs on shutdown                  [LOW]
  File: main.ts:279-297
  Issue: Calls provider.list() and terminates everything. No DB cross-ref.
    Will kill actively-running instances on restart.
  Fix: Only terminate VMs with no matching DB record or in terminal state.
