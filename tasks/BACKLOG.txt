## #BL-15: Agent panic payload sends instance_id, control plane omits it [RESOLVED → #AGENT-01]
  ---

  (lines 266, 289) but control plane HeartbeatPanicStartRequest /
  HeartbeatPanicCompleteRequest types in api/agent.ts omit the field.

  Resolved: dedicated POST /v1/instances/:id/panic route with instance-level auth.

## #BL-16: Idempotency requests should include request timestamp       [PENDING]
  ---
  Retry requests arriving hours after the initial attempt could collide
  with a recycled idempotency key whose TTL has expired and been cleaned.
  Including a client-side request timestamp (e.g. X-Request-Timestamp or
  a field in the body) lets the server reject stale retries outside a
  validity window, preventing weird edge cases with long-delayed retries.
  Likely folds into #API-06 (rate limiting / request validation).
