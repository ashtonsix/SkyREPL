      Ch08 §8.5.6 gap 1: Unknown heartbeat task types SHOULD return 'skipped' receipt
        with reason 'unknown_task_type'. Defensive robustness, not core protocol.
        Location: spec/08_PROVIDERS.txt ~line 870 (heartbeat task requirements)
        Fix: Add one-liner "Unknown task types MUST return a 'skipped' receipt."

      Ch08 §8.5.6 gap 2: Provider MAY perform provider-initiated maintenance
        (e.g. volume cleanup) and include results in providerTasks response array.
        Extension point documented in impl/ L2 but not spec.
        Location: spec/08_PROVIDERS.txt ~line 870
        Fix: Add "Providers MAY include providerTasks in heartbeat response."

      Ch08 §8.5.6 gap 3: Heartbeat response MAY include nextHeartbeat.preferredInterval
        for adaptive scheduling. Optimization hint, not behavioral requirement.
        Location: spec/08_PROVIDERS.txt ~line 870
        Fix: Add "Response MAY include preferredInterval for adaptive scheduling."

      Ch13 §13.3.2 gaps 4-6: Object-backed types (Snapshot, Artifact,
        FeatureInstallation) derive materialized fields from tags_json parsing
        rather than dedicated SQL columns. The spec states converters "MUST map
        between Record and Materialized types preserving all fields" but doesn't
        make the tags_json mechanism explicit. Deterministic from type definitions
        (§13.1.4) + objects table schema (Ch02 §2.2); fully specified in impl/
        db.types.ts.l2.txt §Record to Materialized Converters.
        Location: spec/13_TYPES.txt ~line 530 (materializer section)
        Fix: Add "Object-backed types MUST extract materialized fields from the
        tags_json column of the objects table."
