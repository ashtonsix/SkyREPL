--- FRAMEWORK DECISION: Hono+Zod → ElysiaJS+TypeBox ---

[2026-02-08] Framework evaluation completed. Decision: ElysiaJS + TypeBox.
             Rationale:
             - Higher ceiling: Bun-native, deeper type inference, built-in WebSocket
             - TypeBox schemas ARE JSON Schema: Python agent can consume contracts
               directly, OpenAPI generation without conversion
             - Type deduplication: Static<typeof Schema> replaces 11 manual interfaces
             - SSE is DIY regardless of framework (agent command stream needs
               external push into per-connection ReadableStream)
             - Bun already hard-locked (distributing control as binary, not NPM)
             - Accepted risks: single maintainer, patch-version breaking changes,
               smaller community. Pin versions.
