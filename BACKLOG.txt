  Fix: Remove local constant, use TIMING.DEFAULT_MANIFEST_RETENTION_MS.

  engine.ts:207-213: .catch(console.error). Engine crash leaves workflow
  in 'running' forever. Fix: catch → failWorkflow() with INTERNAL_ERROR.

IMPLEMENTATION (bounded, design already sketched)

EX3: SSE backpressure on slow agent consumers
  Status: OPEN
  Severity: MEDIUM — unbounded memory growth possible
  Context: ReadableStreamDefaultController.enqueue() buffers without limit.
  Fix: track queue depth, drop heartbeats when backlogged, close above threshold.

SM-01: Replace DB polling in AgentBridge with EventEmitter
  Severity: HIGH — 2N queries/sec for N concurrent runs
  Context: agent.ts:596-632 waitForEvent() polls every 1s.
  Fix: In-process EventEmitter fired on state change. ~50 lines.

SM-02: Cap pending command queue
  Severity: HIGH — unbounded memory if agent disconnects
  Context: sse-protocol.ts:81 pendingCommands has no size limit.
  Fix: Cap at 50/instance, FIFO evict, filter stale. ~30 lines.

SM-03: Engine: await promises instead of polling
  Severity: MEDIUM — 100ms latency + unnecessary DB reads
  Context: engine.ts:258-261. Already holds Promise.all.
  Fix: Await existing promises. Only poll on crash recovery. ~20 lines.

SM-07: OrbStackError → extend ProviderError
  Severity: HIGH — retry/fallback machinery dead for only real provider
  Context: orbstack.ts OrbStackError extends Error, not ProviderError.
  Fix: Extend ProviderError, map exit codes to subclasses. ~40 lines.

SM-16: Split db.ts god module
  Severity: HIGH — 1219 lines, 60+ exports, imported everywhere
  Context: CRUD for all 9 resource types + migrations + helpers.
  Fix: Split into db/init, db/helpers, db/{resource}.ts. ~0 new code,
  purely mechanical restructuring. Barrel re-export for compat.

SD-04: Agent lock file for warm pool safety
  Severity: HIGH — duplicate agents on warm pool reuse
  Fix: fcntl.flock() on startup. ~10 lines.

SD-05: Agent heartbeat panic timeout + self-termination
  Severity: HIGH — orphaned agents run indefinitely
  Fix: Implement panic state at 15m, shutdown -h at 20m. ~40 lines.

SM-04: CLI: trust WS status, reduce poll to 30s fallback
  Severity: MEDIUM — double load on control plane
  Fix: ~10 lines in run.ts.

SM-31: terminateOrphanedVMs: cross-ref DB before killing
  Severity: LOW but dangerous on restart
  Fix: Only terminate VMs with no matching record. ~15 lines.
